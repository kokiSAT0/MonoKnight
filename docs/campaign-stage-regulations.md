# キャンペーンステージ レギュレーション一覧

MonoKnight の新キャンペーンは、カードでマス目を移動して塔を登る **出口到達型ダンジョン**へ刷新する。ライトユーザーは低難度ダンジョンで周回成長しながら登り、高難度ダンジョンではローグライク式に毎回初期状態から挑む。実装の一次情報は `Game/DungeonDefinition.swift` とし、既存の `Game/CampaignLibrary+Chapter*.swift` / `Game/CampaignStage.swift` は旧目的地制キャンペーンの移行元として扱う。

## 共通仕様

- 新キャンペーンの基本クリア条件は `completionRule = .dungeonExit(exitPoint:)` とする。
- 各フロアは入口、出口、HP、残り手数、敵、床ギミックを持つ。
- 塔ダンジョン本編は通常の手札/NEXT/山札ドローではなく、基本移動と拾得/報酬カードの回数制インベントリを使う。
- 所持カードは10種類まで。同じカードは種類枠を増やさず、残り使用回数として積む。
- 目的地補充は塔ダンジョン本編では使わない。
- 敵はプレイヤーのカード使用後にターン制で行動する。
- ダンジョン処理順は `床ギミック → 出口判定 → 敵ターン → 失敗判定` とする。
- 低難度ダンジョンでは永続成長を持ち込み可能にし、高難度ダンジョンでは永続成長を無効化または記録/外見のみに限定する。

## ダンジョン構成

| 区分 | テーマ | 成長 | 主なギミック |
|----|--------|------|--------------|
| 低難度 | 基礎塔、鍵扉塔、巡回塔 | 永続成長あり | 出口到達、HP、手数、番兵、巡回兵 |
| 中難度 | 罠塔、ワープ塔、麻痺/誘導塔 | 一部持ち込みまたは一時報酬中心 | ワープ、麻痺、押し出し、報酬選択 |
| 高難度 | 崩れる塔、視界制限塔、複合ランダム塔 | 毎回初期状態 | 床割れ、敵増加、視界制限、ランダム階層 |

今後の開発順と判断基準は [`dungeon-development-roadmap.md`](dungeon-development-roadmap.md) を正本とし、本書は実装済み/実装予定の塔ごとの規定を記録する。

## 初期実装の塔

- `DungeonLibrary.shared` は `基礎塔`、`巡回塔`、`鍵扉塔` を提供する。
- 基礎塔はチュートリアル用の 5×5 / 3 フロア構成とし、出口到達、見張り、ひび割れ床を順に確認する。
- 巡回塔は 9×9 / 3 フロア構成とし、巡回兵の経路読み、すれ違い、巡回兵とひび割れ床の組み合わせを順に確認する。今後追加する基礎塔以外の塔も、特別な理由がない限り9×9を標準にする。
- 鍵扉塔は 9×9 / 3 フロア構成とし、開門マスを鍵、障害物マスを扉として扱う。1F は鍵で直通路を開ける基本、2F は鍵道と遠回りの比較、3F は鍵扉、見張り、床落ちカードの組み合わせを確認する。
- 巡回兵がいるフロアでは、巡回兵の現在地から次に進むマスへの矢印を盤面に常時表示する。
- 塔モードの四角い枠は移動可能マスだけに使い、床落ちカードや床ギミックは枠なしのマーカーとして表示する。
- タイトルの「キャンペーン」入口は塔選択 UI を開き、選んだ塔を1Fから連続で開始する。
- 低難度塔では上下左右1マスの基本移動を常時許可する。基本移動はカードを消費せず、手数、敵ターン、床ギミック、出口判定、手数切れ判定の対象になる。
- 低難度塔はカードなしで始まり、基本移動だけでも出口へ届くが、床落ちカードや報酬カードを使うと 2〜4 手程度短縮できる。
- 1F は基本移動と2マス移動の短縮、2F は見張りの縦射線を避ける迂回、3F はひび割れ床列を迂回するかカードで抜ける判断を主役にする。
- 3F は中央のひび割れ床列を直進すると番兵の危険範囲で HP を失う。迂回は手数がぎりぎりで、2F報酬や床落ちカードを使うと大きく短縮できる。
- フロア間では HP と報酬カードの残り使用回数のみ引き継ぎ、手数、敵、床状態、拾得カードは各フロア開始時にリセットする。
- フロア内カード拾得は、そのマスを踏むと追加手数なしで取得する。拾得カードは1回使い切りで、次フロアへ持ち越さない。
- 拾得/報酬候補には、基本移動と同じ上下左右1マスカードを入れない。
- 1F/2F クリア後は既存移動カードの 3択報酬を1枚選び、3回使用可能なラン中報酬として次フロアへ持ち越す。
- 1F 報酬は `右2`、`上2`、`右桂 (選択)`、2F 報酬は `右連続`、`右2`、`右桂 (選択)` とする。
- 3F クリア後は報酬を出さず、基礎塔クリアとして完了する。
- 巡回塔 1F は横巡回を下側から回り込む練習、2F は2体の巡回兵の周期を見て上側/下側を選ぶすれ違い、3F は巡回網とひび割れ床を報酬カードで抜ける最終階とする。
- 巡回塔 1F 報酬は `上2`、`右2`、`右桂 (選択)`、2F 報酬は `右連続`、`上2`、`右桂 (選択)` とする。
- 巡回塔 3F クリア後は報酬を出さず、巡回塔クリアとして完了する。
- 鍵扉塔 1F 報酬は `右2`、`上2`、`右桂 (選択)`、2F 報酬は `上2`、`右2`、`右上2` とする。
- 鍵扉塔 3F クリア後は報酬を出さず、鍵扉塔クリアとして完了する。
- 失敗時の再挑戦は1Fから始める。
- 永続成長、カード削除、カード強化、補助/攻撃/麻痺/押し出し系報酬は次フェーズ以降で追加する。

## 旧目的地制キャンペーンの扱い

- 既存の全8章/64ステージ目的地制定義は、刷新前キャンペーンとしてコード上に残す。
- 新規実装では `DungeonDefinition` / `DungeonLibrary` を優先し、旧キャンペーン定義へ新要素を追加しない。
- スタンダード、Target Lab、クラシカル系の既存ルールは塔ダンジョン化の影響を受けない。

## QA 観点

- 出口到達でクリアし、目的地数に依存しないこと。
- 塔開始時に手札/NEXT/山札補充が行われず、所持カードが空であること。
- 基本移動で所持カードが変わらず、手数とダンジョンターンだけ進むこと。
- 拾得カードは使用で回数が減り、次フロアへ持ち越されないこと。
- 基礎塔/巡回塔の各フロアで、基本移動だけの代表ルートとカード短縮の代表ルートが成立すること。
- 鍵扉塔の各フロアで、開門マスを踏むと対象扉が通行可能になり、基本移動だけの代表ルートと鍵/カード短縮の代表ルートが成立すること。
- HP 0 または残り手数 0 で失敗すること。
- 出口到達した手は、敵ターンや手数切れより先にクリアすること。
- 敵の危険範囲表示と実際のダメージ判定が一致すること。
- 番兵、巡回兵、見張りの挙動がターンごとに再現可能であること。
- ひび割れ床が 1 回目でひび割れ、2 回目で崩落し、通行不可になること。
- 1F/2F クリア後の報酬カードが3回使用可能な所持カードとして次フロアへ反映され、失敗時の再挑戦では破棄されること。
- 低難度ダンジョンでは永続成長が反映され、高難度ダンジョンでは無効化されること。
- 既存スタンダード、Target Lab、クラシカル系が壊れないこと。
