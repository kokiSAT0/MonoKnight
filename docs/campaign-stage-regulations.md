# キャンペーンステージ レギュレーション一覧

MonoKnight のキャンペーンは、標準ルールと同じ **目的地制** を段階的に学ぶ構成とする。ライトユーザーは通常クリアで新要素へ進み、ヘビーユーザーはスター3コンプリートとハイスコアを狙う。実装の一次情報は `Game/CampaignLibrary+Chapter*.swift` と `Game/CampaignStage.swift`。

## 共通仕様

- 全8章 / 64ステージ構成を維持する。
- 全ステージの盤面は 8×8 に統一する。
- 全ステージの `completionRule` は `.targetCollection(goalCount:)`。
- 手札は5枠、NEXTは3枚、同種カードのスタックあり。
- 表示中の目的地は最大3マスで、どれからでも獲得できる。
- キャンペーンのペナルティ設定は `deadlock 0 / manualRedraw 0 / manualDiscard 1 / revisit 0`。
- 手動引き直しはフォーカスとして扱い、`focusCount` をスコアに反映する。
- `additionalVisitRequirements` と `toggleTilePoints` は使わない。
- アンロックはクリア進行。1-1 のみ初期解放、各章1面は前章8面クリア、章内2面以降は直前ステージクリア。
- スターゲートは使わない。スターは進行条件ではなく、やり込み達成度として扱う。

## 章構成

## 実験場要素の本編配置

- カード群は `standard`、`choice`、`ray`、`warp` をすべてキャンペーン内で扱う。
- 特殊マスは `shuffleHand`、`boost`、`slow`、`nextRefresh`、`freeFocus`、`preserveCard`、`draft`、`overload`、`targetSwap`、`openGate` をすべてキャンペーン内で扱う。
- 初出ステージでは効果を読みやすく単独または少数で配置し、8章で複数効果を組み合わせる。

## スター条件

- ★1: ステージクリア。
- ★2: `twoStarScoreTarget` 以下または未満でクリア。
- ★3: `scoreTarget` 以下または未満でクリア。
- `twoStarScoreTarget` の初期値は、★3ラインを維持したうえで `scoreTarget × 1.25` を 5pt 単位で切り上げる。
- ペナルティ合計条件は新キャンペーン定義では使わない。
- ★2/★3 は通常進行に必要ない。低スコアを目指すやり込み目標とする。

## ステージ紹介

- ステージ開始前は「今回の見どころ」を主役にし、新カード、特殊マス、任意スポーンなどを先に示す。全ステージ共通の「目的地集め」は見どころへ入れず、目的地数は補足表示へ下げる。
- キャンペーン内で初めて登場する要素には `NEW` を付け、ライトユーザーが新要素の増加を感じられるようにする。
- スター条件、過去記録、細かいスコア要素は従来どおり詳細表示へ分ける。

## QA 観点

- 全64ステージが目的地制で開始し、全踏破しなくてもクリアできること。
- 章解放がスター数ではなく、前ステージまたは前章8面のクリアで進むこと。
- 表示中3目的地のどれを先に踏んでも獲得でき、複数通過時は通過順に複数獲得できること。
- 障害物マスが固定スポーン、目的地、ワープカード候補に選ばれないこと。
- 実験場の全カード群と全特殊マスが、キャンペーン内に最低1回以上登場すること。
- 2-1で選択カード、2-2で無料フォーカスマスに触れられ、序盤から新要素の増加を感じられること。
- ステージ開始前に「今回の見どころ」が主表示され、初出要素に `NEW` が付き、既出ステージでも主要ギミックが空にならないこと。
- 開門マスが指定障害物を通常マスへ変え、開門先が障害物として配置されていること。
- 加速マスが盤内では進行方向へ 1 マス追加移動し、盤外・障害物方向では加速せず停止すること。
- 減速マスがその手の残り移動を打ち切り、目的地獲得を妨げないこと。
- フォーカス使用回数がリザルトとスコアに反映され、スターはそのスコア段階で判定されること。
- キャンペーンのプレイ中に、現在スコアと ★2 / ★3 の達成ラインを確認できること。
- ワープタイルとワープカードのあるステージで、目的地ハイライトと移動候補が重なっても視認できること。
- 任意スポーンステージで、スポーン選択前から表示中目的地を確認でき、目的地マスを開始位置として選べないこと。
