# キャンペーンステージ レギュレーション一覧

MonoKnight のキャンペーンは、標準ルールと同じ **目的地制** に刷新した。旧版の「全マス踏破」「二度踏み」「トグル」はキャンペーンのクリア条件から外し、目的地へどう近づくかを学ぶ構成へ置き換える。実装の一次情報は `Game/CampaignLibrary+Chapter*.swift` と `Game/CampaignStage.swift`。

## 共通仕様

- 全8章 / 64ステージ構成を維持する。
- 全ステージの `completionRule` は `.targetCollection(goalCount:)`。
- 手札は5枠、NEXTは3枚、同種カードのスタックあり。
- キャンペーンのペナルティ設定は `deadlock 0 / manualRedraw 0 / manualDiscard 1 / revisit 0`。
- 手動引き直しはフォーカスとして扱い、`focusCount` をスコアとスター条件に反映する。
- `additionalVisitRequirements` と `toggleTilePoints` は使わない。旧ギミックの意図は「戻り道」「迂回」「ルート再判断」として章テーマに移植する。
- アンロックは 1-1 のみ初期解放。各章1面は前章スター12個以上、章内2面以降は直前ステージクリア。

| 章 | テーマ | 目的地数 | 主なデッキ / ギミック |
|----|--------|----------|------------------------|
| 1 | 目的地基礎 | 3〜6 | `kingOnly`、`kingPlusKnightOnly`、`standardLight`、フォーカス導入 |
| 2 | 近距離と戻り | 6〜9 | `kingAndKnightBasic`、選択カード導入前の戻り判断 |
| 3 | カード選択 | 8〜11 | 縦横・斜め・桂馬・全選択カード |

| 7 | ワープタイル | 10〜14 | ワープタイル、障害物、ワープカード入門 |
| 8 | 空間支配 | 10〜14 | 固定ワープ、全域ワープ、障害物、総合演習 |

## スター条件

- ★1: ステージクリア。
- ★2: `secondaryObjective` による追加条件。
  - `finishWithinMoves`
  - `finishWithinSeconds`
  - `finishWithFocusAtMost`
  - `finishWithFocusAtMostAndWithinMoves`
- ★3: `scoreTarget` 以下または未満でクリア。
- ペナルティ合計条件は新キャンペーン定義では使わない。

## QA 観点

- 全64ステージが目的地制で開始し、全踏破しなくてもクリアできること。
- 障害物マスが固定スポーン、目的地、ワープカード候補に選ばれないこと。
- フォーカス使用回数がリザルト、スター条件、スコアに反映されること。
- ワープタイルとワープカードのあるステージで、目的地ハイライトと移動候補が重なっても視認できること。
- 任意スポーンステージで、スポーン選択後に目的地とNEXTが破綻しないこと。
