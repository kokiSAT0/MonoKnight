# MonoKnight Product Specification

`docs/agents_legacy.md` を移行元アーカイブとして参照しつつ、本書を MonoKnight のプロダクト要件とゲーム仕様の自己完結型 Source of Truth とする。App Store 提出に耐える品質で正式リリースすることを前提に、実装判断に必要な仕様をここへ集約する。

## 1. プロダクト概要

- タイトル: `MonoKnight`
- ジャンル: カード × 移動パズル
- 基本盤面: 5×5 グリッド、左下原点、中央 `(2,2)` 開始
- コア体験:
  - 手札スロットは 5 枠
  - 同じカードは重なって待機でき、保持上限は 5 種類
  - 先読みは 3 枚
  - 山札は重みプロファイルに基づく重み付き抽選
- クリア条件: 全 25 マスを踏破するとクリア
- スコア: `手数 × 10 + 所要秒数`
  - 少ないほど上位
  - Game Center ランキングへ送信する

## 2. 正式リリースのスコープ

- 対応端末: `iPhone / iPad`
- 対応 OS: `iOS 16+`
- 画面向き: `Portrait` 固定
- 言語: 日本語
- 盤面: 5×5 固定
- 踏破済マス: グレー表示
- 視点: トップダウン
- 手札 UI: 画面下に 5 スロット、次カード 3 枚の先読みを表示
- ランキング: Game Center の single leaderboard
- 広告: AdMob のインタースティシャルを結果画面でのみ表示
- IAP: 永続アイテム `remove_ads_mk` による広告除去
- 収集/送信:
  - Game Center へのスコア送信
  - 広告配信に伴う最小限の計測
  - IDFA は ATT 許諾時のみ利用
  - ATT または UMP 同意が不足する場合は非パーソナライズ広告を使う

## 3. 非スコープ

- デッキ構築
- 特殊カードの追加拡張を前提とした大規模機能
- 日替わりチャレンジ
- クラウドセーブ
- アカウント連携
- 過度な演出や 3D 表現
- 多言語対応
- ゲーム内チュートリアル動画

## 4. コアルール

### 4.1 盤面と座標

- 盤面座標は `(0...4, 0...4)`
- 原点は左下
- 開始位置は中央 `(2,2)`

### 4.2 カードと移動

- カードで指定された `(dx, dy)` を 1 回適用して移動する
- 盤外へ出るカードはそのターン選択不可
- 選択不可のカードは UI 上で薄表示にする

### 4.3 手札と引き直し

- 手札は 5 スロット
- 同一カードは同一スロットへスタックする
- 先読みは常時 3 枚公開する
- 任意の引き直しは `+2 手`
- 手札スロットに並ぶ全カードが使用不能な場合は手詰まりとみなし、`+3 手` のペナルティ後に全引き直しを行う

### 4.4 クリアとスコア

- 全 25 マスの踏破フラグが `true` になったらクリア
- スコアは `手数 × 10 + 所要秒数`
- ペナルティ加算後の手数を使って計算する
- Game Center に送信するスコアも同じ式を使う

## 5. 標準カード定義

- 合計 16 種
- ナイト型: `(±1, ±2)` の 8 種
- 距離 2 の直線/斜め:
  - `(±2, 0)` と `(0, ±2)`
  - `(±2, ±2)`
  - 合計 8 種
- 山札は初期状態では全カード均等重み
- デッキごとに設定した重みプロファイルから個別重み上書きができる

## 6. 永続化と主要フラグ

- `remove_ads_mk` (`Bool`): 広告除去 IAP 購入状態
- `best_points_5x5` (`Int`): ローカルのベストポイント
- `has_submitted_gc` (`Bool`): 初回スコア送信済みフラグ

## 7. デザイン指針

本作の UI は Apple HIG に準拠しつつ、モノクロ主体のシンプルでモダンな見た目を維持する。

- 配色:
  - 背景 `#000`
  - グリッド白
  - 駒白
  - 踏破済マス `#888`
- アクセントカラー:
  - `accentPrimary` / `accentOnPrimary` など既存テーマを限定利用する
  - 無彩色を基本とし、カテゴリー識別が必要な箇所のみ差し込む
- 補助色:
  - 色だけに依存せず形状、パターン、アイコン差分も併用する
- タイポグラフィ: `SF Pro`
- タップ領域: 44pt 以上
- カード間隔: 8pt から 12pt
- アニメーション: 150ms から 250ms
- ハプティクス: 成功/無効操作で軽く返す
- アクセシビリティ:
  - VoiceOver ラベルでカード方向、使用可否、残り踏破数を伝える
- iPad レイアウト:
  - レギュラー幅では中央寄せと余白を確保する
  - シートやモーダルは大きめに表示して崩れを防ぐ
- 広告:
  - プレイ外のみ表示
  - 誤タップ誘導は禁止
- 同意 UI:
  - 初回起動は「説明 → ATT → UMP」の順
  - 設定画面にプライバシー設定導線を常設
- 広告頻度:
  - 結果画面のみ
  - 最低 90 秒間隔
  - 1 プレイ 1 回までを目安

## 8. 品質前提

- 正式リリースを前提にする
- Swift 初心者でも追える実装を優先する
- iPad でも UI 崩れを起こさない
- 1 プレイを最後まで完走できることを最低条件とする
