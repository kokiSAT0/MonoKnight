# MonoKnight Product Specification

本書を MonoKnight のプロダクト要件と正式リリース範囲の Source of Truth とする。
実装レベルの詳細ルールは `docs/game-rules-handbook.md` を併用する。
`docs/agents_legacy.md` は旧仕様アーカイブであり、現行判断には使わない。
App Store 提出に耐える品質で正式リリースすることを前提に、実装判断に必要な仕様をここへ集約する。

## 1. プロダクト概要

- タイトル: `MonoKnight`
- ジャンル: カード × 移動パズル
- メインコンテンツ: 塔を登る **出口到達型ダンジョン**
- 通常プレイ導線: タイトル画面から `DungeonLibrary.shared` の塔選択だけを開く
- 凍結コンテンツ: スタンダード、ハイスコア、デイリーチャレンジ、Target Lab、クラシカル、旧目的地制キャンペーンは通常導線から外し、互換維持後に段階的に削除する
- 凍結コンテンツのテストは、詳細なゲーム品質保証ではなく、保存互換、ID 変換、ロード、クラッシュ回避の最小確認に限定する
- 基本盤面: 塔ダンジョンはフロア定義ごとの盤面を使う。チュートリアル用の基礎塔は 5×5、基礎塔以外の塔は原則 9×9。左下原点、固定開始時は各盤面の中央開始
- コア体験:
  - 各フロアの出口/階段マスへ到達するとクリアし、次フロアへ進む
  - HP、残り手数、敵の危険範囲、床ギミックを読みながらルートを選ぶ
  - 基本移動、床で拾うカード、フロア間で持ち越す報酬カードを使い分ける
  - 所持カードは10種類まで。同じカードは種類枠を増やさず残り回数として積む
  - 移動可能マスは枠で示し、移動途中に踏むマスは塗りで示す
- 旧目的地制の互換ルール:
  - 手札スロットは 5 枠、先読みは 3 枚、表示中目的地は最大 3 マス
  - 目的地を 12 個獲得するとクリアし、スコアは `移動手数 × 10 + 所要秒数 + フォーカス回数 × 15`
  - 通常導線からは外し、新規拡張しない
- 実験場（凍結/削除候補）:
  - `GameMode.targetLab` はランキング対象外のカード・特殊マス実験場
  - 8×8 盤、目的地 20 個、全部入りカード構成、全特殊マス各 1 個を使い、バランス調整前の挙動確認に使う
  - 特殊マス狙いカードを含め、最寄り特殊マスへ近づく移動候補を試せる
  - カード種類と特殊マス種類の有効/無効設定を保存し、互換維持中の検証用途に限って使う。新規開発は行わず、必要な検証機能は塔ダンジョン側へ移してから削除する
- キャンペーン:
  - 新キャンペーンは、塔を登る **出口到達型ダンジョン**へ刷新する
  - タイトルの通常プレイ入口は `DungeonLibrary.shared` の塔選択を開き、当面は `基礎塔`、`成長塔`、`試練塔` の3本だけをカード表示する。各階一覧は通常の塔選択には出さず、開始可能な区間だけ塔カード内の開始ボタンで選ぶ
  - `基礎塔` は5×5 / 3Fのチュートリアル塔、`成長塔` は9×9 / 20Fの周回成長あり本編塔、`試練塔` は9×9 / 3Fの毎回一から挑むローグライク塔として扱う
  - プレイヤーはカードでマス目を移動し、各フロアの出口/階段マスへ到達すると次フロアへ進む。レイ型など途中マスも踏むカードは、移動途中で階段を踏んだ時点で移動を止めてクリアする
  - 基礎塔など許可された塔では、カードを使わず上下左右1マスへ基本移動できる。基本移動はカードを消費しないが、1手として数え、敵ターン、床ギミック、出口判定、手数制限の対象になる
  - 塔ダンジョンではラン開始時の手札/NEXT/山札ドローを使わず、基本移動と拾得/報酬カードの回数制インベントリで進む
  - 拾得カードは床で拾ったフロア限りの1回使い切り、報酬カードは3回使用可能で残り回数をフロア間で持ち越す。クリア後の報酬選択では、未使用の拾得カードから1枚を報酬カード化して持ち越すこともできる。拾得/報酬候補には基本移動と同じ上下左右1マスカードを入れない
  - 所持カードは10種類まで。同じカードは種類枠を増やさず残り回数として積む
  - 基礎塔と成長塔では同一区間のフロア間で HP と報酬カード残り回数のみ引き継ぎ、手数制限、敵、床状態、拾得カードは各フロア開始時にリセットする。成長塔では同一区間内の前フロア階段マスが次フロアの開始マスになる
  - 基礎塔は基本移動だけでも各フロアの出口へ届くが、拾得カードや報酬カードを使うと 2〜4 手程度短縮できるバランスにする。3F は中央のひび割れ床を直進すると番兵の危険範囲で HP を失い、迂回またはカード突破が得になる
  - 成長塔は、巡回兵、鍵と施錠階段、見える罠、ワープ床、敵警戒、回転見張り、固定の岩/柱、床割れを階ごとに段階的に重ねる。回転見張りは7F前後から短射程で導入し、11F以降で鍵、罠、ワープ、巡回兵と複合する。固定ワープカードは成長塔の拾得/報酬から外し、ワープ要素は床ギミック中心に扱う。旧 `巡回塔`、`鍵扉塔`、`ワープ塔`、`罠塔` の内容は独立塔としては表示せず、成長塔の各階へ統合する
  - 成長塔は20F構成とし、全階に2〜4個程度の固定岩/柱を置いて盤面密度、ルート分岐、敵視線の遮断を作る。岩/柱は `impassableTilePoints` の固定障害物で、鍵や開門では消えない。1-10Fで既存ギミックの単独理解から軽い複合まで、11-20Fで巡回、鍵、罠、ワープ、回転見張り、床割れ、報酬選択をより複合する。階段位置は複数パターンへ分散し、周回時に毎回同じ盤面をなぞる感覚を減らす。1-9F / 11-19F の報酬3択は次フロアの安全、短縮、寄り道のいずれかへ効く候補として扱い、10F / 20F は区切り階として報酬を出さない
  - 成長塔1-8Fは、罠、敵、床割れを増やすのではなく、拾得カードを各5枚前後へ増やして易しく賑やかにする。追加カードは短縮、安全ルート、鍵やカード回収への寄り道に効く位置へ置く
  - 試練塔は `.roguelike` 難度の高難度プロトタイプとし、永続成長を持ち込まず、既存の敵、見える罠、ワープ、床割れ、拾得/報酬カードだけで突破する
  - 基礎塔と成長塔のフロアクリア後は、既存移動カードの 3択から1枚増やす、未使用の拾得カードから1枚を報酬カード化する、所持中の報酬カードを強化する、のいずれか1つを選んで次フロアへ進む。追加/報酬化したカードは通常3回使用可能、強化は対象カードの残り使用回数 +1。所持中の報酬カードは報酬を消費せず、報酬選択前に何枚でも持ち越しから外せる
  - 成長塔は10F区切りの独立ラン区間を持つ。10Fクリアで11F開始が解放され、11Fから開始する場合はHPと報酬カードを初期化する
  - 塔で失敗した場合、基礎塔は1Fから、成長塔は現在の区間開始階から再挑戦する
  - 失敗条件は HP 0、残り手数 0、またはダンジョン固有条件とする
  - 敵はターン制で動き、接触・隣接・射線などの危険範囲へ入るとダメージを受ける
  - 戦闘は回避主役とし、撃破・麻痺・押し出しは突破手段として限定的に扱う
  - 高難度ダンジョンでは床割れ、罠、ワープ、敵増加などを組み合わせ、ローグライク式に毎回初期状態から挑む
  - 成長塔では永続成長、区間開始補助、報酬補正などで周回ごとに登りやすくする
  - 永続成長 v3 は3枝型スキルツリーとし、成長塔の5F / 10F / 15F / 20Fを初回クリアしたときに成長ポイントを各1得る。塔選択では成長塔だけに `5F/10F/15F/20F` の獲得済み/未獲得、開始可能区間、準備/報酬/危険回避の成長枝を表示する。初期候補は「道具袋」「登り支度」「報酬の目利き」「カード温存」「足場読み」「踏み直し」とし、強い効果は前提ノードと10F/15F到達でロックする。基礎塔、試練塔、旧目的地制には適用しない
  - 既存の目的地制キャンペーン定義と `CampaignStageSelectionView` は移行元/旧ルールとして一時保持する。通常導線には出さず、保存互換と必要な検証を確認したうえで段階的に削除する

## 2. 正式リリースのスコープ

- 対応端末: `iPhone / iPad`
- 対応 OS: `iOS 16+`
- 画面向き: `Portrait` 固定
- 言語: 日本語
- 盤面: 塔ダンジョンはフロア定義に従う。チュートリアル用の基礎塔は 5×5、基礎塔以外の塔は原則 9×9
- 凍結コンテンツ: スタンダード、ハイスコア、デイリーチャレンジ、Target Lab、クラシカル、旧目的地制キャンペーンは正式リリースのメイン範囲に含めない
- 視点: トップダウン
- 手札 UI: 塔ダンジョンでは所持カードと残り回数を表示する。旧目的地制では画面下に 5 スロット、次カード 3 枚の先読みを表示する
- ランキング: 塔ダンジョンで必要になるまで Game Center スコア送信をメイン体験の必須要件にしない
- 広告: AdMob のインタースティシャルを結果画面でのみ表示
- IAP: 永続アイテム `remove_ads_mk` による広告除去
- 収集/送信:
  - Game Center へのスコア送信は凍結中の旧モード互換に限定する
  - 広告配信に伴う最小限の計測
  - IDFA は ATT 許諾時のみ利用
  - ATT または UMP 同意が不足する場合は非パーソナライズ広告を使う

- 特殊カードの追加拡張を前提とした大規模機能
- 日替わりチャレンジ
- スタンダード/ハイスコア/Target Lab/クラシカル/旧目的地制キャンペーンの新規拡張
- クラウドセーブ
- アカウント連携
- 過度な演出や 3D 表現
- 多言語対応
- ゲーム内チュートリアル動画

## 4. コアルール

### 4.1 盤面と座標

- 盤面座標は各モードの `boardSize` に従う。5×5 は `(0...4, 0...4)`、8×8 は `(0...7, 0...7)`
- 原点は左下
- 固定開始位置は `BoardGeometry.defaultSpawnPoint` が返す中央マス

### 4.2 カードと移動

- カードで指定された `(dx, dy)` を 1 回適用して移動する
- 盤外へ出るカードはそのターン選択不可
- 選択不可のカードは UI 上で薄表示にする

### 4.3 手札とフォーカス

- 手札は 5 スロット
- 同一カードは同一スロットへスタックする
- 先読みは常時 3 枚公開する
- 任意の全引き直しは廃止し、代わりに `フォーカス` を使う
- 任意スポーンの目的地制では、開始位置を選ぶ前から手札・先読み・表示中目的地を見られる
- 任意スポーンで目的地マスを選んだ場合は、開始位置にできないことをトーストで知らせる
- フォーカスは表示中の目的地へ近づきやすいカードを優先して手札と先読みを再構成する
- フォーカス使用時は手数を増やさず、スコアへ `+15` を加算する
- 手札スロットに並ぶ全カードが使用不能な場合は、ペナルティではなく目的地へ近づく再配布を自動実行する

### 4.4 クリアとスコア

- 表示中の目的地に到達すると `capturedTargetCount` が増え、クリア前は表示中目的地が最大 3 マスになるよう補充される
- 同じ手の経路上で複数の表示中目的地を通過した場合は、通過順に複数獲得できる
- 目的地を 12 個獲得したら即クリアし、終盤に余って表示されていた目的地は追加ノルマとして数えない
- 踏破済みマスは内部履歴として保持するが、目的地制では通常マスのグレー塗り分けを表示せず、スタンダードモードのクリア条件にも使わない
- スコアは `移動手数 × 10 + 所要秒数 + フォーカス回数 × 15`
- Game Center に送信するスコアも同じ式を使う

### 4.5 塔ダンジョン

- 塔ダンジョンでは `GameMode.CompletionRule.dungeonExit(exitPoint:)` を使い、指定出口マスへ到達するとフロアクリアになる。途中マスも踏むカードでは、解錠済みの出口/階段に入った時点で移動を止める。
- ダンジョン追加ルールは `DungeonRules` に集約し、難度、HP/手数制限、敵定義、床ギミック、基本移動の可否をフロア単位で差し替える。
- 敵は `EnemyDefinition` と `EnemyState` で管理し、番兵、巡回兵、見張り、回転見張りを扱う。回転見張りはプレイヤーの1手ごとに視線方向を順番に切り替え、現在の危険範囲と次方向矢印を盤面上で読めるようにする。
- 敵の危険範囲は UI 表示とダメージ判定の共通情報として扱い、表示と実際の被弾がずれないようにする。
- 床割れは `HazardDefinition.brittleFloor` として扱い、1 回踏むとひび割れ、2 回目で崩落して通行不可になる。塔ダンジョンでは崩落ごとに HP を 1 減らし、報酬や成長を挟まず下階の同じマスへ落下する。落下先の同じマスが無傷のひび割れ床なら着地時にひび割れて停止し、既にひび割れている場合だけさらに崩落して連鎖落下する。落下で HP 0 になった場合は失敗扱いにし、下階へ移らない。成長塔では区切り階や最終階など、通常遷移できる次階がない階にひび割れ床を置かない。
- ダメージ罠は `HazardDefinition.damageTrap` として扱い、踏んだ罠マスごとに指定ダメージを受ける。罠は最初から見える表示にし、隠し罠、麻痺、押し出しは v1 では追加しない。
- ダンジョン処理順は、移動解決後に `床ギミック → 出口判定 → 敵ターン → 失敗判定` とする。
- 成長塔など `DungeonRules.exitLock` を持つフロアでは、鍵マスを踏むまで出口/階段は施錠中になり、到達してもクリアしない。成長塔の鍵は階段ロック解除専用で、未取得時だけ鍵アイコンを出し、取得後は盤面から消える。移動途中で鍵マスを踏んだ場合も取得済みになり、鍵取得後は短い解錠演出を出し、階段到達で通常どおりクリアする。`TileEffect.openGate(target:)` は旧目的地制、Target Lab、Daily、ヘルプ/辞典、保存互換向けの開門特殊マスとして残す。
- 成長塔と試練塔では `DungeonFloorDefinition.warpTilePairs` から既存の `TileEffect.warp` を塔ダンジョン内で再利用する。固定ワープカードは試練塔や旧モード互換では使えるが、成長塔では拾得/報酬に混ぜない。
- 成長塔の固定岩/柱は `DungeonFloorDefinition.impassableTilePoints` で定義する通行不可マスとして扱う。spawn / exit / 鍵 / 拾得カード / 敵初期位置 / 罠 / ひび割れ床 / ワープ床とは重ねず、基本移動だけの代表導線を残す。移動候補、レイ型カード、敵視線、ワープ候補の既存通行不可判定に従う。
- 成長塔の拾得カードとクリア後報酬候補はランごとの seed で変化する。同じラン中は固定し、位置は spawn / exit / 鍵 / 敵 / 罠 / 通行不可 / ワープ上を避ける。報酬候補は各階の基準候補を役割の正本とし、seed 変化は同系統カードへの差し替えに留める。Standard / Classical / Target Lab / Daily のカード候補はこの変化の対象外とする。

## 5. 標準カード定義

- 合計 16 種
- ナイト型: `(±1, ±2)` の 8 種
- 距離 2 の直線/斜め:
  - `(±2, 0)` と `(0, ±2)`
  - `(±2, ±2)`
  - 合計 8 種
- 目的地制の山札は初期状態では全カード均等重み
- デッキごとに設定した重みプロファイルから個別重み上書きができる

## 6. 永続化と主要フラグ

- `remove_ads_mk` (`Bool`): 広告除去 IAP 購入状態
- `best_points_5x5` (`Int`): ローカルのベストポイント
- `has_submitted_gc` (`Bool`): 初回スコア送信済みフラグ
- `dungeon_growth_v3` (`JSON`): 成長塔の永続成長ポイント、取得済みスキルツリーノード、5F/10F/15F/20Fの初回報酬済み区切りID、解放済みチェックポイント開始階。リリース前整理として旧 `dungeon_growth_v2` は読み込まない

## 7. デザイン指針

本作の UI は Apple HIG に準拠しつつ、モノクロ主体のシンプルでモダンな見た目を維持する。

- 配色:
  - 背景 `#000`
  - グリッド白
  - 駒白
  - 踏破済マス `#888` はクラシカルなど全踏破モードの進捗表示に限定
- アクセントカラー:
  - `accentPrimary` / `accentOnPrimary` など既存テーマを限定利用する
  - 無彩色を基本とし、カテゴリー識別が必要な箇所のみ差し込む
- 補助色:
  - 色だけに依存せず形状、パターン、アイコン差分も併用する
- タイポグラフィ: `SF Pro`
- タップ領域: 44pt 以上
- カード間隔: 8pt から 12pt
- アニメーション: 150ms から 250ms
- ハプティクス: 成功/無効操作で軽く返す
- アクセシビリティ:
  - VoiceOver ラベルでカード方向、使用可否、残り目標数を伝える
- iPad レイアウト:
  - レギュラー幅では中央寄せと余白を確保する
  - Portrait ではゲーム盤面を適度な上限内で大きく表示し、手札・NEXT・上部操作との視線移動を抑える
  - タイトル、塔選択、ゲーム中、リザルトは広い横幅を活かしつつ、本文が横に伸びすぎない最大幅へ収める
  - シートやモーダルは大きめに表示して崩れを防ぐ
- 広告:
  - プレイ外のみ表示
  - 誤タップ誘導は禁止
- 同意 UI:
  - 初回起動は「説明 → ATT → UMP」の順
  - 設定画面にプライバシー設定導線を常設
- 広告頻度:
  - 結果画面のみ
  - 6 回クリアごとに 1 回まで
  - 最低 90 秒間隔
  - 1 プレイ 1 回までを目安

## 8. 品質前提

- 正式リリースを前提にする
- Swift 初心者でも追える実装を優先する
- iPad でも UI 崩れを起こさない
- 1 プレイを最後まで完走できることを最低条件とする
