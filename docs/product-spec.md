# MonoKnight Product Specification

本書を MonoKnight のプロダクト要件と正式リリース範囲の Source of Truth とする。
実装レベルの詳細ルールは `docs/game-rules-handbook.md` を併用する。
`docs/agents_legacy.md` は旧仕様アーカイブであり、現行判断には使わない。
App Store 提出に耐える品質で正式リリースすることを前提に、実装判断に必要な仕様をここへ集約する。

## 1. プロダクト概要

- タイトル: `MonoKnight`
- ジャンル: カード × 移動パズル
- メインコンテンツ: 塔を登る **出口到達型ダンジョン**
- 通常プレイ導線: タイトル画面から `DungeonLibrary.shared` の塔選択だけを開く
- 削除済みコンテンツ: スタンダード、ハイスコア、デイリーチャレンジ、Target Lab、クラシカル、旧目的地制キャンペーンは通常導線と実コードから外す。現行仕様として扱わない
- 基本盤面: 塔ダンジョンはフロア定義ごとの 9×9 盤面を使う。左下原点、固定開始時は各盤面の中央開始
- コア体験:
  - 各フロアの出口/階段マスへ到達するとクリアし、次フロアへ進む
  - HP、敵の危険範囲、床ギミックを読みながらルートを選び、残り手数は迷走防止の目安として確認する
  - プレイ中の状態異常は上部情報欄にミニアイコンと添え字で常時表示し、詳細説明はアイコンをタップした時だけモーダルシートで表示する
  - 基本移動、床で拾うカード、フロア間で持ち越す報酬カードを使い分ける
  - 所持カードは通常カード9種類までにし、10枠目は消費されない基本移動カードとして固定表示する。同じカードは種類枠を増やさず残り回数として積む
  - 移動可能マスは枠で示し、移動途中に踏むマスは塗りで示す
- 塔攻略:
  - MonoKnight の通常プレイは、塔を登る **出口到達型ダンジョン**へ集約する
  - タイトルの通常プレイ入口は `DungeonLibrary.shared` の塔選択を開き、当面は `基礎塔`、`成長塔`、`試練塔` の3本だけをカード表示する。各階一覧は通常の塔選択には出さず、開始可能な区間だけ塔カード内の開始ボタンで選ぶ
  - 塔フロア開始時は、説明カードや開始ボタンを挟まず、塔名と階数だけの短い開始演出を表示して自動的に盤面へ移る。演出中のタップは準備完了後だけ即時開始として扱う
  - `基礎塔` は9×9 / 6Fのチュートリアル塔、`成長塔` は9×9 / 50Fの周回成長あり本編塔、`試練塔` は9×9 / 無限階の毎回一から挑むローグライク塔として扱う
  - プレイヤーはカードでマス目を移動し、各フロアの出口/階段マスへ到達すると次フロアへ進む。レイ型など途中マスも踏むカードは、移動途中で階段を踏んだ時点で移動を止めてクリアする
  - 基礎塔など許可された塔では、10枠目に固定表示される基本移動カードから上下左右1マスへ基本移動できる。基本移動はカードを消費しないが、1手として数え、敵ターン、床ギミック、出口判定、手数制限の対象になる
  - 塔ダンジョンではラン開始時の手札/NEXT/山札ドローを使わず、基本移動と拾得/報酬カードの回数制インベントリで進む
  - 拾得カードは床で拾った1回カードとして扱い、クリア時に未使用分が残っていれば次フロアへそのまま持ち越す。移動報酬カードは通常2回使用可能で、残り回数をフロア間で持ち越す。上下左右1マスカードと上下/左右選択キングは廃止済みで、上下左右1マスは基本移動のみが担当する
  - 補助カード `補給` は移動せず1手使って自身を消費し、通常カード枠が9種類まで埋まるように、塔用移動カード全体から未所持の一時移動カードを1回分ずつ補給する。呪文系補助カード `消滅の呪文` は手札で選んでから敵1体を指定し、移動せず1手使ってその敵だけを消滅させる。`全滅の呪文` は移動せず1手使って自身を消費し、現在フロアの敵をすべて消滅させる。`凍結の呪文` は敵ターンを3回止める。`障壁の呪文` は使用直後から3回分の行動後処理でHPダメージだけを無効化する。`ダークネスの呪文` はその階の見張り/回転見張りのレーザーだけを封じる。`レール破壊の呪文` はその階の巡回兵のレール移動だけを封じる。`解毒薬` は毒状態を解除し、`万能薬` は毒・足枷・幻惑状態を解除する。解毒薬と万能薬は解除対象がない時は使えず、ダークネスの呪文は見張り系がいない階や発動済みの階、レール破壊の呪文は巡回兵がいない階や発動済みの階では使えない。補助カードは効果解決後にその場でターン経過し、溶岩滞在ダメージと敵ターンの対象になる。補助カードは床拾得にもクリア後報酬にも低確率で混ざり、カード枠を使う1回カードとして扱う
  - 所持カードは通常カード9種類までにし、10枠目は消費されない基本移動カードとして固定表示する。同じカードは種類枠を増やさず残り回数として積む
  - 基礎塔、成長塔、試練塔では連続進行するフロア間で HP と所持カード残り回数を引き継ぎ、成長塔では挑戦中だけ有効な遺物も引き継ぐ。手数制限、敵、床状態、床落ちカード配置は各フロア開始時にリセットする。どの塔でも、連続して次フロアへ進む場合は前フロアの階段マスが次フロアの開始マスになる
  - 基礎塔は基本移動だけでも各フロアの出口へ届くが、拾得カードや報酬カードを使うと 2〜4 手程度短縮できるバランスにする。1Fで出口到達とカード拾得、2Fで見張りの射線、3Fで番兵とひび割れ床、4Fで鍵と施錠出口、5Fで見える罠、6Fでワープ床と短い巡回兵を順に学ぶ。3F は中央のひび割れ床を直進すると番兵の危険範囲で HP を失い、迂回またはカード突破が得になる。基礎塔未クリアで成長塔を初回開始する時だけ基礎塔を勧めるが、成長塔自体はロックしない
  - 成長塔は、巡回兵、鍵と施錠階段、見える罠、ワープ床、敵警戒、回転見張り、追跡兵、メテオ兵、暗闇、固定の岩/柱、床割れ、回復マス、沼を階ごとに段階的に重ねる。巡回兵は9F以降で6〜8マス程度の広い巡回へ段階拡大し、待つ/すれ違う/カードで越す判断を増やす。回転見張りは7F前後から固定岩/柱で視線を遮れる形で導入し、11F以降で鍵、罠、ワープ、巡回兵と複合する。追跡兵は7F以降に少量導入し、待ちすぎ、寄り道、鍵回収の遅れでHPを削られる圧を作る。メテオ兵と暗闇は17F以降に少量だけ導入し、制御ランダムな着弾予告や見張り射線を読みながら近くの情報だけで進む終盤判断にする。回復マスは中盤以降に少量だけ置き、踏むと HP が 1 増えて消費され、危険な近道の前後で一度だけ立て直せる。沼は12F前後から複数マスの塊として置き、移動系カードの抜け道を途中で止めるが、基本移動と補給は使える軽い地形制限として扱う。ワープカードは退役し、ワープ要素は床ギミック中心に扱う。旧 `巡回塔`、`鍵扉塔`、`ワープ塔`、`罠塔` の内容は独立塔としては表示せず、成長塔の各階へ統合する
  - 成長塔は50F構成とし、全階に2〜4個程度の固定岩/柱を置いて盤面密度、ルート分岐、敵視線の遮断を作る。岩/柱は `impassableTilePoints` の固定障害物で、鍵を取っても消えない。1-10Fで基本要素を顔見せし、11-20Fで暗闇、メテオ、宝箱、沼、手札喪失系を少量混ぜる。21-30Fは鍵、宝箱、ワープ、巡回待ち、追跡圧による経路選択、31-40Fは毒、足枷、幻惑、暗闇、回復マス、補助カードによる状態異常と情報制限、41-50Fは長い巡回、回転見張り、メテオ、落下、呪い宝箱、報酬ビルドの総合試験として扱う。階段位置は複数パターンへ分散し、周回時に毎回同じ盤面をなぞる感覚を減らす。1-49F の報酬3択は次フロアの安全、短縮、寄り道のいずれかへ効く候補として扱い、10F/20F/30F/40Fは成長区切りだが報酬を選んで次区間へ進める。50F は最終階として報酬を出さず、登頂リザルトで塔累計手数、残HP、最終手札、所持レリック、所持呪いをまとめて見せる
  - 成長塔1-8Fは、罠、敵、床割れを増やすのではなく、拾得カードを各5枚前後へ増やして易しく賑やかにする。追加カードは短縮、安全ルート、鍵やカード回収への寄り道に効く位置へ置く
  - 成長塔には数階に1個の宝箱を置き、踏むとランダムに遺物を即取得する。遺物はカード枠を消費しない挑戦中だけの効果で、取得済み宝箱は同じラン中に再取得できず盤面から消える。同じ通常遺物/呪い遺物は1回の挑戦中に重複して出ない。遺物は床拾得には混ぜず、宝箱またはクリア報酬からのみ取得する。宝箱やクリア報酬で通常遺物/呪い遺物/ミミック/補填結果が出た時は、軽い取得演出と詳細表示を挟み、確認タップ後に宝箱なら同じフロアへ戻り、報酬なら次フロアへ進む。基礎塔と試練塔には宝箱と遺物を入れない
  - 宝箱は通常宝箱と怪しい宝箱に分ける。通常宝箱は未所持の通常遺物だけを出し、怪しい宝箱は盤面上で見た目と VoiceOver 名を変え、低確率でミミック、パンドラの箱、呪い遺物を出す。通常遺物にはコモン、レア、レジェンドの希少度を持たせ、通常宝箱、怪しい宝箱、クリア報酬、手早いクリア補正でレア/レジェンドの出やすさを変える
  - 遺物と呪い遺物はゲームオーバー、区間開始からの再挑戦、新規開始、最終クリアで失われる。中断復帰では所持遺物、所持呪い遺物、取得済み宝箱を保持する。有限塔の最終クリア時だけ、失われる前の所持遺物と呪い遺物を今回のランの記録として登頂リザルトに表示する
  - 所持遺物と呪い遺物はプレイ画面下部の手札の下にアイコンで横並び表示し、タップで名前、短い効果、補足や消費条件を確認できる。通常遺物はコモン/レア/レジェンドの希少度を表示し、有限回数の効果は残り回数で示す。一時/永続の分類名や `一` / `永` の文字バッジは出さず、警告風の代償表示も出さない。呪い遺物だけ利点/代償/解除条件を分けて表示する
  - 試練塔は `.roguelike` 難度の高難度塔とし、挑戦開始時に永続成長、スキルツリー効果、開始カード、成長補正、遺物を持ち込まない。ラン中だけ HP、拾得/報酬カード、累計手数を持ち越し、各階の生成 seed と階数から9×9フロアを決定論的に生成する。1Fから数階は軽めにしつつ、階数が上がるほど罠、敵、床割れ、ワープ、暗闇、状態異常床を段階的に増やす
  - 基礎塔と成長塔のフロアクリア後は、移動/補助カードまたは低確率の遺物から報酬を1つ選んで次フロアへ進む。手札に追加した移動カードは通常2回使用可能、補助カードは通常1回使用可能、遺物はカード枠を使わずラン中の所持遺物へ追加する。報酬候補が既に所持しているカードと同種だった場合は、種類枠を増やさず残り使用回数として積む。所持中の報酬カードは報酬を消費せず、報酬選択前に何枚でも手札から外せる
  - 成長塔は10F/20F/30F/40Fクリアで次区間開始を解放する。区切り階クリア時は通常報酬を選んで同じランのまま次区間へ進めるが、解放済み区間から新規開始する場合はHPと報酬カードを初期化する
  - 塔で失敗した場合、基礎塔と試練塔は1Fから、成長塔は現在の区間開始階から再挑戦する。成長塔の復帰系スキルは再挑戦時だけ追加支給として効き、新規開始や通常の区間開始は強めない。試練塔の再挑戦は新しい生成 seed の新規ランとして扱う
  - 失敗条件は HP 0 またはダンジョン固有条件とする。残り手数 0 の後も続行できるが、上限超過1手目、その後は超過4手目、7手目、10手目...で防御/障壁では防げない疲労ダメージを1受ける。手数上限は通常ルートに余裕を持たせ、だらだらプレイ防止のための疲労開始ラインとして扱う。残り手数が0になると、手数ゲージ領域は3つの丸による疲労インジケーターへ切り替わり、安全な手数消費中ではないことを示す
  - 敵はターン制で動き、接触・隣接・射線などの危険範囲へ入るとダメージを受ける
  - 戦闘は回避主役とし、撃破・麻痺・押し出しは突破手段として限定的に扱う
  - 高難度ダンジョンでは床割れ、罠、ワープ、敵増加などを組み合わせ、ローグライク式に毎回初期状態から挑む
  - 成長塔では永続成長、区間開始補助、報酬補正などで周回ごとに登りやすくする
  - 永続成長 v3 は50F拡張を見据えた分岐網型スキルツリーとし、成長塔の5F刻み初回クリアで成長ポイントを各1得る。10F/20F/30F/40F/50Fなど区間終端の再クリアでも周回ポイントを得られ、周回すれば最終的に全スキルを取得・同時有効化できる。成長塔カード内では成長塔だけに5F刻みの獲得済み/未獲得、開始可能区間、準備/報酬/危険回避/索敵/復帰の役割カードと選択系統の前提スキル順タイムラインを表示し、「全体」表示で5系統の段階ノード図も確認できる。スキル取得条件はポイントと前提スキル解放だけで、到達階層は成長ポイント獲得、区間開始解放、効果発動階層にだけ使う。索敵系が有効な場合は、同じカード内の「次区間の見通し」に床ギミック、報酬、敵、経路、深層/踏破の傾向を挑戦前の短文として表示する。復帰系は成長塔の失敗後再挑戦だけに効き、21F以降の補給/障壁、31F以降の万能薬、41F以降の経路カード/凍結を追加支給する。取得済みスキルはコストなしで個別にオン/オフでき、効果判定は有効中のスキルだけを参照する

## 2. 正式リリースのスコープ

- 対応端末: `iPhone / iPad`
- 対応 OS: `iOS 16+`
- 画面向き: `Portrait` 固定
- 言語: 日本語
- 盤面: 塔ダンジョンはフロア定義に従う。基礎塔、成長塔、試練塔はいずれも原則 9×9
- 現行コンテンツ: 塔ダンジョンのみ
- 視点: トップダウン
- 手札 UI: 塔ダンジョンでは所持カードと残り回数を表示する
- ランキング: 塔ダンジョンで必要になるまで Game Center スコア送信をメイン体験の必須要件にしない
- 広告: AdMob のインタースティシャルを結果画面でのみ表示
- IAP: 永続アイテム `remove_ads_mk` による広告除去
- 収集/送信:
  - Game Center は将来の試練塔ランキング用に認証・送信・leaderboard 表示の薄いサービス境界だけを残す。現時点の塔プレイからは呼ばない
  - 広告配信に伴う最小限の計測
  - IDFA は ATT 許諾時のみ利用
  - ATT または UMP 同意が不足する場合は非パーソナライズ広告を使う

- アカウント連携

- 多言語対応
- ゲーム内チュートリアル動画

## 4. コアルール

### 4.1 盤面と座標

- 盤面座標は各フロアの `boardSize` に従う。現行の塔は9×9で `(0...8, 0...8)` を使う
- 原点は左下
- 固定開始位置は `BoardGeometry.defaultSpawnPoint` が返す中央マス

### 4.2 カードと移動

- カードで指定された `(dx, dy)` を 1 回適用して移動する
- 盤外へ出るカードはそのターン選択不可
- 選択不可のカードは UI 上で薄表示にする

### 4.3 所持カードと報酬

- 塔ダンジョンでは初期手札/NEXT/山札補充を使わず、床で拾うカードとクリア後報酬カードを所持品として扱う。
- 拾得カードは1回使い切りだが、クリア時に未使用分が残っていれば報酬カードと同じ所持品として区間内フロア間へ持ち越す。
- 所持カードは通常カード9種類までにし、基本移動が許可された塔では10枠目を消費されない基本移動カードとして固定表示する。同じカードは種類枠を増やさず、残り使用回数として積む。
- 報酬選択では移動/補助カード追加、低確率の未所持遺物取得から1つ選ぶ。移動/補助/遺物は同じ候補枠に並び、成長効果なしでは最大3択、広い見立て、`勝利の旗`、`割れた羅針盤` の補正込みで最大4択にする。報酬候補が既に所持しているカードと同種だった場合は、種類枠を増やさず残り使用回数として積む。
- 試練塔の報酬候補は成長塔のスキル補正を受けず、生成階数に応じた最大3択として扱う。

### 4.4 クリアと失敗

- 塔ダンジョンでは `GameMode.CompletionRule.dungeonExit(exitPoint:)` を使い、指定出口マスへ到達するとフロアクリアになる。
- 途中マスも踏むカードでは、解錠済みの出口/階段に入った時点で移動を止める。
- HP 0 で失敗する。残り手数 0 の後も続行できるが、上限超過1手目、その後は超過4手目、7手目、10手目...で疲労ダメージを1受ける。疲労ダメージは防御/障壁では防げない。各フロアの代表ルートには余裕を持たせ、だらだらプレイ防止の疲労開始ラインとして扱う。残り手数が0になると、手数ゲージ領域は3つの丸による疲労インジケーターへ切り替わる。
- 失敗リザルトでは「盤面を見る」から失敗時点の盤面を確認でき、結果へ戻るかホームへ戻るかを選べる。
- 出口到達した手は、敵ターンや疲労判定より先にクリアする。

### 4.5 塔ダンジョン詳細

- 塔ダンジョンでは `GameMode.CompletionRule.dungeonExit(exitPoint:)` を使い、指定出口マスへ到達するとフロアクリアになる。途中マスも踏むカードでは、解錠済みの出口/階段に入った時点で移動を止める。
- ダンジョン追加ルールは `DungeonRules` に集約し、難度、HP/手数制限、敵定義、床ギミック、基本移動の可否をフロア単位で差し替える。
- 敵は `EnemyDefinition` と `EnemyState` で管理し、番兵、巡回兵、見張り、回転見張り、追跡兵、メテオ兵を扱う。敵本体は種類ごとに形の違う盤面マーカーで表示し、遊び方内の敵辞典で同じ見た目と読み方を確認できる。敵は種類を問わず、プレイヤーがそのマスを踏むと倒れる。レイ型、吹き飛ばし、ワープなどで通過したマスも踏んだ扱いにし、踏んだ敵はその手の敵ターンでは攻撃しない。敵同士は同じマスに重ならず、敵ターンでは先に処理された敵がいる/予約したマスへ後続の敵は進まない。吹き飛ばしマスは表示矢印の固定方向へ、盤外または障害物に当たる直前まで移動し、通過マスの踏破・敵撃破・罠・鍵・出口・拾得カードは通常移動と同じ処理にする。巡回兵は既存の `patrol(path:)` を使い、成長塔中盤以降では短い2×2巡回だけでなく6〜8マス程度の巡回路も使う。巡回路は薄い灰色のレールとして常時表示し、次に進む方向は巡回兵アイコン内の向きで表示する。回転見張りは4方向固定で、プレイヤーの1手ごとに右回りまたは左回りへ視線方向を切り替え、回転後に向いている方向へ障害物/壁に当たるまでの直線上すべてを危険範囲として扱う。盤面の赤い射線は次の敵ターンで実際に被弾する範囲として表示し、回転見張りの次方向は盤面上の別矢印ではなく、敵アイコンの瞳孔部分にある回転矢印から読む。追跡兵はプレイヤーの1手後に通行可能な最短経路で1マス近づき、表示する危険範囲は上下左右1マスにする。敵本体マスは敵マーカーとして分け、追跡兵の次移動先も盤面上の小さな矢印で常時表示する。巡回兵と追跡兵は、プレイヤーが移動後に現在地からの危険範囲へ入っている場合は移動せずその場で攻撃し、現在地では届かない場合だけ移動後の危険範囲で攻撃する。メテオ兵は動かず、通行可能な床から制御ランダムに選んだ複数マスを着弾予告マスとして表示し、次の敵ターンでそのマスにいると1ダメージを与える。凍結の呪文が有効な間は、敵本体マーカーだけを残して危険範囲、着弾予告、次移動プレビューを消し、敵ターンの移動、回転、攻撃、メテオ着弾を合計3回分スキップする。障壁の呪文が有効な間は、敵の移動、回転、メテオ更新、危険範囲、着弾予告、次移動プレビューを通常どおり残したまま、敵/メテオによるHPダメージだけを無効化する。ダークネスの呪文が有効な階では、見張りと回転見張りの射線危険と射線ダメージだけを消し、敵本体、回転、その他の敵や床効果は通常どおり残す。レール破壊の呪文が有効な階では、巡回兵のレールと次移動プレビューを消し、巡回兵の移動だけを止める。巡回兵本体、隣接危険、隣接/接触ダメージは残す。プレイヤー行動後は短い敵ターン演出を挟み、敵の移動、危険/警告マス更新、攻撃結果を順に見せてから次の入力へ戻す。
- 敵の危険範囲表示は敵本体マスを含めず、周囲/射線/警告対象など実際に攻撃されるマスだけを示す。敵本体マスは敵マーカーとして表示し、プレイヤーが入ると撃破する対象として扱う。
- レイ型などの連続移動で敵の危険範囲を通過した場合は、そのマスへ入った時点で先にHPダメージを受ける。その後で敵本体マスを踏めば敵は倒れ、その敵は以後の敵ターンでは攻撃しない。
- 床割れは `HazardDefinition.brittleFloor` として扱い、初期状態を3種類に分ける。`cracked` は目視できるヒビ床で、踏むと崩落床になるがその踏みでは落下しない。`hiddenWeak` は見た目が通常床の高難易度床で、踏むと崩落床になるがその踏みでは落下しない。`collapsed` は最初から崩れた穴で、入ると即座に HP を 1 減らし、報酬や成長を挟まず前の階の同じマスへ落下する。落下時は短い落下演出の後、開始画面を挟まず前の階の盤面へ戻る。落下先の同じマスがヒビ床または隠し脆い床なら崩落床に変わって停止し、既に崩落床の場合だけさらに連鎖落下する。落下で HP 0 になった場合や、前の階がない場合は失敗扱いにし、別の階へ移らない。敵の移動、危険範囲、着弾予告は崩落穴を避ける。成長塔では区切り階や最終階など、通常遷移できる次階がない階に床割れを置かない。成長塔では 1-10F はヒビ床のみ、11-25F はヒビ床と少数の崩落床、26F以降は隠し脆い床を少量混ぜる。成長塔後半の一部では、落下でしか入れない小部屋に宝箱を置き、帰還ワープで通常エリアへ戻れる任意の寄り道報酬として扱う。
- ダメージ罠は `HazardDefinition.damageTrap` として扱い、踏んだ罠マスごとに指定ダメージを受ける。溶岩は `HazardDefinition.lavaTile` として扱い、踏んだ時に指定ダメージを受け、さらに溶岩上で補助カードなどの移動しない行動を使うと補助効果後に追加ダメージを受けてから敵ターンへ進む。回復マスは `HazardDefinition.healingTile` として扱い、初めて踏んだ時に HP が 1 増えて消費される。HP 上限は設けず、レイ型、吹き飛ばし、ワープ先など途中マスも踏んだ扱いになる移動で発動する。障壁の呪文が有効な間は、ダメージ罠、溶岩、毒、崩落穴のHPダメージも無効化するが、床破壊や落下などの非HP効果は発生する。麻痺罠は既存の `TileEffect.slow` 枠をプレイヤー向けに置き換えた見える床ギミックとして扱い、踏むとしびれて1回休みになり、レイ型移動はそのマスで止まる。足枷罠は `TileEffect.shackleTrap` として扱い、踏むと次の階まで足枷状態になり、手数消費は通常どおりだが敵ターンが最大2フェーズ進む。足枷は万能薬で解除できる。毒罠は `TileEffect.poisonTrap` として扱い、踏んだ手ではダメージを受けず移動も止まらないが、次の成功行動から3行動ごとに最大3回 HP を失う。毒は解毒薬または万能薬で解除できる。幻惑罠は `TileEffect.illusionTrap` として扱い、踏むとその階の間だけ移動カードが `？` 表示になり、押した時に現在合法な移動カードと移動先からランダムに解決する。基本移動と補助カードは通常どおり使え、幻惑は万能薬または次フロア開始で解除される。沼は `TileEffect.swamp` として扱い、移動系カードで入るとそこで止まり、沼の上では移動系カードを使えない。基本移動と補助カードは沼上でも使える。カード喪失罠はランダム1枠、移動カード全枠、補助カード全枠、全手札枠の4効果を持ち、移動後に対象カード枠を失わせる。移動/補助のカテゴリ喪失は中盤以降、全手札喪失は深層または試練寄りのレア罠として扱う。隠し罠、押し出しは v1 では追加しない。
- ダンジョン処理順は、移動解決後に `床ギミック → 出口判定 → 毒処理 → 敵ターン → 失敗判定` とする。毒罠を踏んだ手だけは、その手の毒処理をスキップする。
- 成長塔など `DungeonRules.exitLock` を持つフロアでは、鍵マスを踏むまで出口/階段は施錠中になり、到達してもクリアしない。成長塔の鍵は階段ロック解除専用で、未取得時だけ鍵アイコンを出し、取得後は盤面から消える。移動途中で鍵マスを踏んだ場合も取得済みになり、鍵取得後は短い解錠演出を出し、階段到達で通常どおりクリアする。
- 成長塔と試練塔では `DungeonFloorDefinition.warpTilePairs` から既存の `TileEffect.warp` を塔ダンジョン内で再利用する。`全域ワープ` と `固定ワープ` は退役カードとし、塔攻略の拾得/報酬には混ぜない。
- 試練塔では `DungeonRunState` に保持した生成 seed と現在階数からフロアを再生成する。同じ seed と同じ階数では中断復帰後も同じ配置になり、違う seed または違う階数では別配置になる。移動不能マスだけで入口から出口までの上下左右経路が詰む盤面は生成せず、入口、出口、鍵、敵、罠、拾得カード、通行不可、ワープを重ねない。見張りの初期向きは、盤外や岩/柱ですぐ止まる方向を避け、同じ位置から有効射線が最も長い方向へ寄せる。階層表示は `試練塔 12F` のように分母なしで表示し、ローカル最高到達階を `UserDefaults` に保存して塔選択とリザルトに表示する。Game Center 送信は行わない。
- 成長塔の固定岩/柱は `DungeonFloorDefinition.impassableTilePoints` で定義する通行不可マスとして扱う。spawn / exit / 鍵 / 拾得カード / 敵初期位置 / 罠 / 床割れ / ワープ床とは重ねず、基本移動だけの代表導線を残す。移動候補、レイ型カード、敵視線、ワープ候補の既存通行不可判定に従う。
- 暗闇フロアは `DungeonFloorDefinition.isDarknessEnabled` で指定し、判定は通常通りのまま盤面表示だけを制限する。常時見えるのは階段、鍵、見張り/回転見張りのレーザー危険マス、メテオ着弾予告マスで、敵本体、拾得カード、宝箱、罠、床効果、巡回レールなどはプレイヤー現在地を含む周囲1マスに入った時だけ表示する。凍結中やダークネスの呪文で既存通り危険表示が消える場合は、その消灯を優先する。
- 成長塔の拾得カード、敵初期位置/向き/巡回路、鍵位置、ワープ位置、階段接続、岩/柱、床ギミック、宝箱位置、クリア後報酬候補はランごとの seed で制御ランダムに変化する。同じラン中は固定し、中断復帰や落下で同じ階へ戻っても同じ配置になる。各階の敵種、敵数、鍵の有無、ワープペア数、階テーマ、手数上限は固定テンプレートを維持する。見張り/回転見張りの初期向きは、盤外や岩/柱ですぐ止まる方向を避け、同じ位置から有効射線が最も長い方向へ寄せる。階段は区切り階を含めて近傍候補へ揺れ、前階の階段位置が次階の開始位置になる。拾得カード数は既存数を基準に -1 / 0 / +1 程度で揺れ、1-8F は4〜6枚、9F以降は最低3枚を保つ。ランダム配置は spawn / exit / 鍵 / 敵 / 巡回路 / 罠 / 通行不可 / ワープ / 拾得カード / 宝箱上を避け、鍵あり階では開始地点から鍵、鍵から階段まで基本移動だけの代表導線を残す。ワープは必須ルートではなく選択肢として扱い、ワープなしでも階段へ到達できる。カード種別は 1-5F、6-10F、11-15F、16-20F、21-30F、31-40F、41-50F の階層帯と床拾得/クリア報酬ごとの重み付きプールから抽選する。解毒薬は6F以降、万能薬、ダークネスの呪文、レール破壊の呪文は11F以降の補助枠に低確率で混ぜる。床拾得は移動約90%/補助約10%で、遺物は出さない。クリア報酬は通常時に移動約89%/補助約10%/遺物約1%で、手数が上限の70%以下なら 77%/20%/3%、50%以下なら 65%/30%/5% へ寄せる。
- 成長塔の宝箱は `DungeonFloorDefinition.relicPickups` で定義し、spawn / exit / 鍵 / 敵 / 罠 / 通行不可 / ワープ / 拾得カードと重ねない。移動途中で踏んだ場合も拾得カードと同じ移動解決中に取得するが、カード所持枠は消費しない。取得結果は移動演出後に詳細表示し、確認タップで同じフロアの操作へ戻る。落下専用小部屋の宝箱も同じ取得済み管理に含め、同じラン中に再取得できない。12Fは怪しい宝箱の軽量版、16F/19Fは怪しい宝箱の深層版にする。通常宝箱で未所持通常遺物が尽きた場合は重複付与せず、HP +1 の小補填だけ行って宝箱を消す。通常宝箱の通常遺物抽選は コモン75 / レア22 / レジェンド3、怪しい宝箱・軽量は 55 / 35 / 10、怪しい宝箱・深層は 40 / 42 / 18 を使う。クリア報酬のレリック枠は通常時 70 / 25 / 5、70%以内クリアで 60 / 32 / 8、50%以内クリアで 45 / 40 / 15 に寄せ、`曇った鏡` がある場合は手早いクリアによる希少度補正を無効化する。
- 通常遺物は48種とし、直接的なマイナス効果を持たない強化アイテムとして扱う。有限回数を持つ通常遺物は残り回数を表示し、回数なしで挑戦中続く通常遺物には分類名を表示しない。通常宝箱、怪しい宝箱、パンドラ、クリア報酬では既に所持している通常遺物を候補から除外する。既存24種のうち `不滅の心臓` は各フロア開始時 HP +1 とする。既存24種に加え、`木彫りの護符` は取得時 HP +1、`銅の砂時計` は手数上限 +2、`旅の保存食` は各フロア開始時 HP2以下なら HP +1、`小さなランタン` は暗闇視界 +1、`鈍い針` は次の罠ダメージを1回だけ1軽減、`継ぎ接ぎロープ` は次の落下HP減少を1回だけ1軽減、`野戦医療箱` は回復マスの回復量 +1、`斥候の羅針盤` は70%以内クリア時に報酬候補 +1、`早業の鞘` は非レイ型移動報酬カードの使用回数 +1、`清めの護符` は次の状態異常または手札喪失系を1回だけ無効化、`不死鳥の羽根` は致死ダメージを1回だけ HP1 で耐える、`賢者の写本` は新規の拾得カード、移動報酬カード、補助報酬カードの使用回数 +1、`罠踏みの靴底` はダメージ罠から受けるHPダメージを常に1軽減、`残り火の外套` は溶岩と溶岩滞在から受けるHPダメージを常に1軽減、`見張りの単眼鏡` は見張り・回転見張りから受けるHPダメージを1軽減、`レール守りの護符` は巡回兵から受けるHPダメージを1軽減、`追跡避けの囮` は追跡兵から受けるHPダメージを1軽減、`解毒石` は毒罠の毒ダメージ回数を1減らして最低1回、`星除けの傘` はメテオと標的警告から受けるHPダメージを1軽減、`落下止めの錨` は崩落穴の落下HP減少を常に1軽減、`折りたたみ地図` は鍵を拾って出口を開けた階の手数上限 +2、`幻の切符` はワープマス移動時に使用した移動カードを消費しない、`焚き火の熾火` は回復マスで毒・足枷・幻惑を解除、`商人の天秤` はクリア報酬でレリックを選んだ時に次階開始HP +1とする。
- 怪しい宝箱は箱ごとの危険度で抽選する。軽量版は通常遺物75% / 呪い遺物15% / ミミック7% / パンドラ3%、深層版は通常遺物60% / 呪い遺物25% / ミミック10% / パンドラ5%。通常遺物、呪い遺物、パンドラの候補が成立しない場合はその結果を抽選テーブルから外し、残りの重みで再正規化する。`ミミック` は即時に HP -1 し、赤い杯などでHP0になれば失敗する。`パンドラの箱` は未所持の通常遺物1つと未所持の呪い遺物1つを同時に得る。

## 5. カード定義

### 発見型辞典

- 遊び方内の辞典は、カード、敵、マス、レリック、イベントを発見型コレクションとして扱う。通常時は遭遇済み項目だけ詳細を表示し、未遭遇項目は名称、見た目、効果を `？？？` として隠す。
- 遭遇済み判定は「盤面、手札、床拾得、報酬候補、取得結果としてプレイヤー画面に表示された時点」で記録する。敵や罠は踏まなくても、画面で見えた時点で辞典から読み返せる。
- プレイ中に補助カードや表示中の特殊マスを長押しすると、辞典と同じ名称・効果説明を小さなその場表示で確認できる。通常床は長押し説明の対象にしない。
- 開発者向け診断メニューが有効な環境では、設定から「辞典を全表示（開発者用）」をオンにでき、発見状態に関係なく全項目を確認できる。この設定は確認用で、通常プレイヤー向けの発見率は変更しない。

- 塔攻略では `MoveCard` の移動カードを、拾得カード、報酬カード、補助的な固定フロア設定として使う。
- 標準移動カード、選択式カード、レイ型カード、ワープ系カードは `Deck.Configuration` やフロア定義から参照できる。
- 成長塔の拾得/報酬候補は、短縮、安全ルート、寄り道の役割が分かる現行カードにする。上下左右1マスカードと上下/左右選択キングは使わない。

## 6. 永続化と主要フラグ

- `remove_ads_mk` (`Bool`): 広告除去 IAP 購入状態
- `gc_has_submitted_by_leaderboard` / `gc_last_score_by_leaderboard`: 将来の Game Center leaderboard 用の送信記録
- `dungeon_growth_v3` (`JSON`): 成長塔の永続成長ポイント、取得済みスキルツリーノード、有効中スキルツリーノード、5F刻みの初回報酬済み区切りID、解放済みチェックポイント開始階。`activeUpgrades` がない旧 v3 保存は取得済みスキルをすべて有効として扱う。リリース前整理として旧 `dungeon_growth_v2` は読み込まない
- `dungeon_run_resume_v1` (`JSON`): 塔攻略中断復帰用の単一保存枠。現在の塔ID、階、`DungeonRunState`、現在位置、手数、経過秒、HP、敵、鍵、拾得/報酬カード、所持遺物、所持呪い遺物、取得済み宝箱、ひび割れ/崩落床を保存し、塔選択の「続きから」から直近の一手後へ復帰する。タイトルへ戻る明示操作では保持し、失敗、最終階/区切り階クリア、リセット、新規開始時は破棄する
- `encyclopedia_discovery_v1` (`[String]`): 遊び方辞典の発見済み項目 ID。`category:itemID` 形式で保存し、未知 ID は無視して将来項目追加に備える。
- `shows_all_encyclopedia_entries_for_developer` (`Bool`): 開発者向け診断メニュー内の辞典全表示トグル。発見状態そのものは変更しない。

## 7. デザイン指針

本作の UI は Apple HIG に準拠しつつ、モノクロ主体のシンプルでモダンな見た目を維持する。

- 配色:
  - 背景 `#000`
  - グリッド白
  - 駒白
  - 踏破済マス `#888` は塔攻略で必要な履歴表示に限定
- アクセントカラー:
  - `accentPrimary` / `accentOnPrimary` など既存テーマを限定利用する
  - 無彩色を基本とし、カテゴリー識別が必要な箇所のみ差し込む
- 補助色:
  - 色だけに依存せず形状、パターン、アイコン差分も併用する
- 回数制限付きレリックは、残り 0 回になってもプレイ中の詳細確認は維持し、グレー化と残り 0 表示で使用済みを示す
- タイポグラフィ: `SF Pro`
- タップ領域: 44pt 以上
- カード間隔: 8pt から 12pt
- 手札の方向ソートでは移動カードを先、補助カードを後ろにまとめ、移動カード内を移動方向順に並べる
- アニメーション: 150ms から 250ms
- ハプティクス: 成功/無効操作で軽く返す
- プレイ中フィードバック: 移動完了、無効操作、被ダメージ、カード/宝箱取得、階段解放は短いパルスや揺れで即時に伝える
- 使用不可カードをタップした時は、短い非モーダル表示で使えない理由を返す
- アクセシビリティ:
  - VoiceOver ラベルでカード方向、使用可否、残り目標数を伝える
- iPad レイアウト:
  - レギュラー幅では中央寄せと余白を確保する
  - Portrait ではゲーム盤面を適度な上限内で大きく表示し、手札・NEXT・上部操作との視線移動を抑える
  - タイトル、塔選択、ゲーム中、リザルトは広い横幅を活かしつつ、本文が横に伸びすぎない最大幅へ収める
  - シートやモーダルは大きめに表示して崩れを防ぐ
- 広告:
  - プレイ外のみ表示
  - 誤タップ誘導は禁止
- 同意 UI:
  - 初回起動は「説明 → ATT → UMP」の順
  - 設定画面にプライバシー設定導線を常設
- 広告頻度:
  - 結果画面のみ
  - 10 回クリアごとに 1 回まで
  - 最低 90 秒間隔
  - 1 プレイ 1 回までを目安
