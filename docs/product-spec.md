# MonoKnight Product Specification

本書を MonoKnight のプロダクト要件と正式リリース範囲の Source of Truth とする。
実装レベルの詳細ルールは `docs/game-rules-handbook.md` を併用する。
`docs/agents_legacy.md` は旧仕様アーカイブであり、現行判断には使わない。
App Store 提出に耐える品質で正式リリースすることを前提に、実装判断に必要な仕様をここへ集約する。

## 1. プロダクト概要

- タイトル: `MonoKnight`
- ジャンル: カード × 移動パズル
- メインコンテンツ: 塔を登る **出口到達型ダンジョン**
- 通常プレイ導線: タイトル画面から `DungeonLibrary.shared` の塔選択だけを開く
- 凍結コンテンツ: スタンダード、ハイスコア、デイリーチャレンジ、Target Lab、クラシカル、旧目的地制キャンペーンは通常導線から外し、互換維持後に段階的に削除する
- 基本盤面: 塔ダンジョンはフロア定義ごとの盤面を使う。チュートリアル用の基礎塔は 5×5、基礎塔以外の塔は原則 9×9。左下原点、固定開始時は各盤面の中央開始
- コア体験:
  - 各フロアの出口/階段マスへ到達するとクリアし、次フロアへ進む
  - HP、残り手数、敵の危険範囲、床ギミックを読みながらルートを選ぶ
  - 基本移動、床で拾うカード、フロア間で持ち越す報酬カードを使い分ける
  - 所持カードは10種類まで。同じカードは種類枠を増やさず残り回数として積む
  - 移動可能マスは枠で示し、移動途中に踏むマスは塗りで示す
- 旧目的地制の互換ルール:
  - 手札スロットは 5 枠、先読みは 3 枚、表示中目的地は最大 3 マス
  - 目的地を 12 個獲得するとクリアし、スコアは `移動手数 × 10 + 所要秒数 + フォーカス回数 × 15`
  - 通常導線からは外し、新規拡張しない
- 実験場（凍結/削除候補）:
  - `GameMode.targetLab` はランキング対象外のカード・特殊マス実験場
  - 8×8 盤、目的地 20 個、全部入りカード構成、全特殊マス各 1 個を使い、バランス調整前の挙動確認に使う
  - 特殊マス狙いカードを含め、最寄り特殊マスへ近づく移動候補を試せる
  - カード種類と特殊マス種類の有効/無効設定を保存し、互換維持中の検証用途に限って使う。新規開発は行わず、必要な検証機能は塔ダンジョン側へ移してから削除する
- キャンペーン:
  - 新キャンペーンは、塔を登る **出口到達型ダンジョン**へ刷新する
  - タイトルの通常プレイ入口は `DungeonLibrary.shared` の塔選択を開き、基礎塔、巡回塔、鍵扉塔を1Fから連続で開始する
  - チュートリアル用の基礎塔は5×5、巡回塔以降の基礎塔以外の塔は原則9×9の盤面で構成する
  - プレイヤーはカードでマス目を移動し、各フロアの出口/階段マスへ到達すると次フロアへ進む
  - 基礎塔など許可された塔では、カードを使わず上下左右1マスへ基本移動できる。基本移動はカードを消費しないが、1手として数え、敵ターン、床ギミック、出口判定、手数制限の対象になる
  - 塔ダンジョンではラン開始時の手札/NEXT/山札ドローを使わず、基本移動と拾得/報酬カードの回数制インベントリで進む
  - 拾得カードは床で拾ったフロア限りの1回使い切り、報酬カードは3回使用可能で残り回数をフロア間で持ち越す。拾得/報酬候補には基本移動と同じ上下左右1マスカードを入れない
  - 所持カードは10種類まで。同じカードは種類枠を増やさず残り回数として積む
  - 低難度塔ではフロア間で HP と報酬カード残り回数のみ引き継ぎ、手数制限、敵、床状態、拾得カードは各フロア開始時にリセットする
  - 基礎塔は基本移動だけでも各フロアの出口へ届くが、拾得カードや報酬カードを使うと 2〜4 手程度短縮できるバランスにする。3F は中央のひび割れ床を直進すると番兵の危険範囲で HP を失い、迂回またはカード突破が得になる
  - 巡回塔は巡回兵の経路読みを主役にする低難度塔とし、1F/2F報酬や床落ちカードで巡回ルートを安全に抜ける代表ルートを作る
  - 鍵扉塔は既存の開門マスを鍵、障害物マスを扉として使い、鍵を取って施錠中の階段を有効化してから出口へ向かう流れを主役にする
  - 低難度塔の 1F/2F クリア後は、既存移動カードの 3択報酬を1枚選び、そのカードを3回使用可能なラン中報酬として次フロアへ持ち越す
  - 低難度塔で失敗した場合、再挑戦は1Fから始める
  - 失敗条件は HP 0、残り手数 0、またはダンジョン固有条件とする
  - 敵はターン制で動き、接触・隣接・射線などの危険範囲へ入るとダメージを受ける
  - 戦闘は回避主役とし、撃破・麻痺・押し出しは突破手段として限定的に扱う
  - 高難度ダンジョンでは床割れ、視界制限、敵増加などを組み合わせ、ローグライク式に毎回初期状態から挑む
  - 低難度ダンジョンでは永続成長、スキルツリー、初期デッキ補正などで周回ごとに登りやすくする
  - 低難度塔の永続成長 v1 は選択式ツリーとし、塔を最後まで初回クリアしたときに成長ポイントを 1 得る。塔選択では、低難度塔ごとの成長ポイント獲得済み/未獲得を表示する。初期実装では「初期HP +1」と「報酬候補強化」の 2 種だけを扱い、高難度ローグライクや旧目的地制には適用しない
  - 既存の目的地制キャンペーン定義と `CampaignStageSelectionView` は移行元/旧ルールとして一時保持する。通常導線には出さず、保存互換と必要な検証を確認したうえで段階的に削除する

## 2. 正式リリースのスコープ

- 対応端末: `iPhone / iPad`
- 対応 OS: `iOS 16+`
- 画面向き: `Portrait` 固定
- 言語: 日本語
- 盤面: 塔ダンジョンはフロア定義に従う。チュートリアル用の基礎塔は 5×5、基礎塔以外の塔は原則 9×9
- 凍結コンテンツ: スタンダード、ハイスコア、デイリーチャレンジ、Target Lab、クラシカル、旧目的地制キャンペーンは正式リリースのメイン範囲に含めない
- 視点: トップダウン
- 手札 UI: 塔ダンジョンでは所持カードと残り回数を表示する。旧目的地制では画面下に 5 スロット、次カード 3 枚の先読みを表示する
- ランキング: 塔ダンジョンで必要になるまで Game Center スコア送信をメイン体験の必須要件にしない
- 広告: AdMob のインタースティシャルを結果画面でのみ表示
- IAP: 永続アイテム `remove_ads_mk` による広告除去
- 収集/送信:
  - Game Center へのスコア送信は凍結中の旧モード互換に限定する
  - 広告配信に伴う最小限の計測
  - IDFA は ATT 許諾時のみ利用
  - ATT または UMP 同意が不足する場合は非パーソナライズ広告を使う

## 3. 非スコープ

- 特殊カードの追加拡張を前提とした大規模機能

- スタンダード/ハイスコア/Target Lab/クラシカル/旧目的地制キャンペーンの新規拡張
- クラウドセーブ
- アカウント連携
- 過度な演出や 3D 表現
- 多言語対応
- ゲーム内チュートリアル動画

## 4. コアルール

### 4.1 盤面と座標

- 盤面座標は各モードの `boardSize` に従う。5×5 は `(0...4, 0...4)`、8×8 は `(0...7, 0...7)`
- 原点は左下
- 固定開始位置は `BoardGeometry.defaultSpawnPoint` が返す中央マス

### 4.2 カードと移動

- カードで指定された `(dx, dy)` を 1 回適用して移動する
- 盤外へ出るカードはそのターン選択不可
- 選択不可のカードは UI 上で薄表示にする

### 4.3 手札とフォーカス

- 手札は 5 スロット
- 同一カードは同一スロットへスタックする
- 先読みは常時 3 枚公開する
- 任意の全引き直しは廃止し、代わりに `フォーカス` を使う
- 任意スポーンの目的地制では、開始位置を選ぶ前から手札・先読み・表示中目的地を見られる
- 任意スポーンで目的地マスを選んだ場合は、開始位置にできないことをトーストで知らせる
- フォーカスは表示中の目的地へ近づきやすいカードを優先して手札と先読みを再構成する
- フォーカス使用時は手数を増やさず、スコアへ `+15` を加算する
- 手札スロットに並ぶ全カードが使用不能な場合は、ペナルティではなく目的地へ近づく再配布を自動実行する

### 4.4 クリアとスコア

- 表示中の目的地に到達すると `capturedTargetCount` が増え、クリア前は表示中目的地が最大 3 マスになるよう補充される
- 同じ手の経路上で複数の表示中目的地を通過した場合は、通過順に複数獲得できる
- 目的地を 12 個獲得したら即クリアし、終盤に余って表示されていた目的地は追加ノルマとして数えない
- 踏破済みマスは内部履歴として保持するが、目的地制では通常マスのグレー塗り分けを表示せず、スタンダードモードのクリア条件にも使わない
- スコアは `移動手数 × 10 + 所要秒数 + フォーカス回数 × 15`
- Game Center に送信するスコアも同じ式を使う

### 4.5 塔ダンジョン

- 塔ダンジョンでは `GameMode.CompletionRule.dungeonExit(exitPoint:)` を使い、指定出口マスへ到達するとフロアクリアになる。
- ダンジョン追加ルールは `DungeonRules` に集約し、難度、HP/手数制限、敵定義、床ギミック、基本移動の可否をフロア単位で差し替える。
- 敵は `EnemyDefinition` と `EnemyState` で管理し、初期実装では番兵、巡回兵、見張りを扱う。
- 敵の危険範囲は UI 表示とダメージ判定の共通情報として扱い、表示と実際の被弾がずれないようにする。
- 床割れは `HazardDefinition.brittleFloor` として扱い、1 回踏むとひび割れ、2 回目で崩落して通行不可になる。
- ダンジョン処理順は、移動解決後に `床ギミック → 出口判定 → 敵ターン → 失敗判定` とする。
- 鍵扉塔のように `DungeonRules.exitLock` を持つフロアでは、鍵マスを踏むまで出口/階段は施錠中になり、到達してもクリアしない。鍵取得後は短い解錠演出を出し、階段到達で通常どおりクリアする。

## 5. 標準カード定義

- 合計 16 種
- ナイト型: `(±1, ±2)` の 8 種
- 距離 2 の直線/斜め:
  - `(±2, 0)` と `(0, ±2)`
  - `(±2, ±2)`
  - 合計 8 種
- 目的地制の山札は初期状態では全カード均等重み
- デッキごとに設定した重みプロファイルから個別重み上書きができる

## 6. 永続化と主要フラグ

- `remove_ads_mk` (`Bool`): 広告除去 IAP 購入状態
- `best_points_5x5` (`Int`): ローカルのベストポイント
- `has_submitted_gc` (`Bool`): 初回スコア送信済みフラグ
- `dungeon_growth_v1` (`JSON`): 塔ダンジョンの永続成長ポイント、取得済みアップグレード、初回報酬済み塔 ID

## 7. デザイン指針

本作の UI は Apple HIG に準拠しつつ、モノクロ主体のシンプルでモダンな見た目を維持する。

- 配色:
  - 背景 `#000`
  - グリッド白
  - 駒白
  - 踏破済マス `#888` はクラシカルなど全踏破モードの進捗表示に限定
- アクセントカラー:
  - `accentPrimary` / `accentOnPrimary` など既存テーマを限定利用する
  - 無彩色を基本とし、カテゴリー識別が必要な箇所のみ差し込む
- 補助色:
  - 色だけに依存せず形状、パターン、アイコン差分も併用する
- タイポグラフィ: `SF Pro`
- タップ領域: 44pt 以上
- カード間隔: 8pt から 12pt
- アニメーション: 150ms から 250ms
- ハプティクス: 成功/無効操作で軽く返す
- アクセシビリティ:
  - VoiceOver ラベルでカード方向、使用可否、残り目標数を伝える
- iPad レイアウト:
  - レギュラー幅では中央寄せと余白を確保する
  - Portrait ではゲーム盤面を適度な上限内で大きく表示し、手札・NEXT・上部操作との視線移動を抑える
  - タイトル、塔選択、ゲーム中、リザルトは広い横幅を活かしつつ、本文が横に伸びすぎない最大幅へ収める
  - シートやモーダルは大きめに表示して崩れを防ぐ
- 広告:
  - プレイ外のみ表示
  - 誤タップ誘導は禁止
- 同意 UI:
  - 初回起動は「説明 → ATT → UMP」の順
  - 設定画面にプライバシー設定導線を常設
- 広告頻度:
  - 結果画面のみ
  - 6 回クリアごとに 1 回まで
  - 最低 90 秒間隔
  - 1 プレイ 1 回までを目安

## 8. 品質前提

- 正式リリースを前提にする
- Swift 初心者でも追える実装を優先する
- iPad でも UI 崩れを起こさない
- 1 プレイを最後まで完走できることを最低条件とする
