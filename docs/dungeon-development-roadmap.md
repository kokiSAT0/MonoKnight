# 塔ダンジョン開発ロードマップ

MonoKnight の当面の主軸は、カードでマス目を移動して塔を登る **出口到達型ダンジョン**を育てることに置く。
通常ユーザー向けのプレイ導線は塔ダンジョンへ集約し、スタンダード、ハイスコア、デイリーチャレンジ、Target Lab、クラシカル、旧目的地制キャンペーンは通常導線と実コードから削除済みの旧コンテンツとして扱う。

本ロードマップは、塔ダンジョンを `基礎塔 / 成長塔 / 試練塔` の3本立てで磨き、それぞれ「学ぶ」「周回で伸びる」「毎回一から挑む」役割を明確にするための判断基準である。

## 1. 開発の主軸

- 新要素は広げすぎず、常に「カードで移動し、敵、床、出口を読む」判断密度を増やす方向に寄せる。
- 敵は基本的に倒す対象ではなく、避ける対象として扱う。
- 報酬は単純な強化より、ルート選択や危険回避の方針が変わるものを優先する。
- UI は派手さより、出口、危険範囲、報酬効果、失敗理由がすぐ分かることを優先する。
- App Store 向けの TestFlight QA、広告、IAP、Game Center、ATT/UMP、本番値整備は、ゲーム内容の再開発とは別レーンで管理する。

## 2. フェーズ

### Phase 1: 基礎塔の手触り固め

- 3F 連続ラン、HP 引き継ぎ、基本移動、床落ちカード、3択カード報酬の体験を調整する。
- 拾得カード配置、報酬候補、敵配置、手数制限、床割れ位置を見直す。
- 「選んだ報酬が次階で効いた」と感じられることを優先する。
- 基本移動で詰まりを減らし、拾得カードは一時的な短縮、報酬カードは複数回使えるルート方針として価値を持たせる。
- 拾得カード、報酬カード、基本移動はリザルトでも役割が分かれるようにし、床カードと報酬カードを混ぜて表示しない。
- 失敗時に、なぜ失敗したか、次にどう変えるかが分かる表示を整える。

### Phase 2: 成長塔を本編化する

- 既存の巡回、施錠階段、ワープ、罠の内容は、独立塔として並べず 20F 構成の `成長塔` へ段階的に統合する。成長塔の鍵は階段ロック解除専用にし、障害物扉の開門は旧目的地制/実験系特殊マスとして切り分ける。
- 成長塔は、1-10F で巡回、鍵、見える罠、ワープ床、敵警戒、回転見張り、追跡兵、固定の岩/柱、床割れを単独理解から軽い複合まで学び、11-20F で複数要素を同時に読む本編塔にする。固定岩/柱は全20Fに2〜4個程度配置し、開門で消える扉ではなく、ルート分岐、視線遮断、盤面密度を作る軽量な障害物として扱う。巡回兵は9F以降で6〜8マス程度へ段階拡大し、待つ/すれ違う/カードで越す判断を増やす。回転見張りは7F前後から短射程で導入し、次方向矢印を読んで待つ/抜ける/カードで飛ぶ判断を作る。追跡兵は7F以降に少量導入し、待ちすぎ、寄り道、鍵回収の遅れに対してHPが削られる負け筋を作る。固定ワープカードは成長塔から外し、特定座標ワープの学習は旧目的地制/試練塔/実験系へ分離する。
- 10F クリアで `11F開始` を解放し、1-10F と 11-20F はそれぞれ独立したラン区間として扱う。HP と報酬カードの引き継ぎは区間内だけ維持する。
- 長期的には 50F 以上を目指すが、階層追加は 10F 単位で、既存要素の組み合わせと読める難しさを確認しながら進める。

### Phase 3: 永続成長を成長塔攻略へ結びつける

- 成長塔向けに、周回で少しずつ登りやすくなる成長を追加する。
- 成長ポイントは、成長塔の 5F / 10F / 15F / 20F を初回クリアしたときに得る。同じ区切りでは重複付与しない。
- 初期候補は3枝型スキルツリーにする。準備枝は道具袋、登り支度、報酬枝は報酬の目利き、カード温存、危険回避枝は足場読み、踏み直しに絞る。
- 塔選択では、成長塔だけに `5F/10F/15F/20F` の成長ポイント獲得済み/未獲得と、開始可能区間が分かるようにする。基礎塔は「チュートリアル」、試練塔は「成長なし」として役割を短く示す。
- 報酬の目利きは、毎回同じ強候補へ偏らせず、既存候補を補完する方向に混ぜる。
- 勝てないものを数値で押し切るより、開始条件、報酬選択、危険回避の判断が変わる成長にする。
- 基礎塔と試練塔では永続成長を無効化、または記録/見た目のみに限定する。

### Phase 4: 報酬とデッキ構築を深める

- 現在のカード追加から、カード削除、カード強化、一時スキルへ広げる。
- v1 では、カード追加3択を残しつつ、所持中の報酬カードは報酬選択として「使用回数 +1」を選べる。手札から外す整理は報酬を消費せず、次フロアへ進む前に何枚でも行える。
- 攻撃力強化ではなく、移動、回避、誘導、危険回避の癖が出るビルドにする。
- 補助カード、麻痺、押し出し系はこの段階で検討する。
- 説明しにくい割に効果が薄いものは入れない。

### Phase 5: 中難度/高難度へ拡張する

- 中難度は罠、ワープ、麻痺/誘導など、ギミック理解を主役にする。
- 高難度は床割れ、視界制限、敵増加、ランダム階層、固定シード/日替わりルールを組み合わせる。
- 高難度はローグライク式に毎回初期状態から始め、報酬と一時ビルドだけで勝負する。
- `試練塔` は最小の高難度プロトタイプとして追加済み。永続成長を使わず、既存の敵、罠、ワープ、床割れを組み合わせる入口にする。

### 高難度塔プロトタイプの制約

- 高難度塔は毎回初期状態から開始し、成長塔の「周回で登りやすくなる」体験とは分離する。
- 永続成長は強さへ影響させない。表示や記録として扱う場合も、初期 HP、報酬候補、カード性能には適用しない。
- 勝敗はラン中報酬、一時ビルド、フロアごとの判断で決まるようにする。
- v0 は固定3フロアで開始し、ランダム階層、日替わり、視界制限、追加敵種は手触り確認後に検討する。

## 3. 直近のおすすめ順

1. 成長塔20Fの実プレイ調整
2. 成長塔30F化に向けた階層設計
3. 報酬強化/持ち越し整理の実プレイ調整
4. 広告頻度と同意 UI の UX 検証
5. 試練塔の実プレイ調整

## 4. 継続ルール

- スタンダード、Target Lab、クラシカル、デイリー、旧目的地制キャンペーンは通常導線へ戻さない。
- 削除済み旧コンテンツの保存データ移行や互換 UI は新規に追加しない。
- 旧コンテンツ前提のテストは復活させない。
- 削った旧テストが守っていた観点を塔攻略でも必要と判断した場合は、旧テストを復活させず、塔ダンジョン文脈のテストとして書き直す。
- 仕様変更が入ったら `product-spec.md`、`campaign-stage-regulations.md`、`game-mode-parameters.md` を同期する。
- 機能追加ごとに `swift test`、関連 app tests、iOS Simulator build、`git diff --check` を通す。
- 触ったファイルで局所的な整理が必要な場合だけ、機能追加と同じ変更内で小さく整理する。
