# MonoKnight ゲームルールハンドブック

本書は MonoKnight に実装されているゲームルールの詳細 Source of Truth とする。
プロダクト要件は [`product-spec.md`](product-spec.md) を併用し、旧仕様アーカイブの [`agents_legacy.md`](agents_legacy.md) は現行判断に使わない。
現行メインコンテンツは塔攻略のみであり、旧目的地制キャンペーン、Daily、ハイスコア、旧標準/クラシカル/Target Lab は現行仕様として扱わない。

## 1. 盤面・座標・初期配置

- 盤面は `BoardGeometry` が提供する N×N グリッド。塔フロアは `DungeonFloorDefinition` ごとの盤面サイズを使う。
- 基礎塔は 5×5、成長塔と試練塔は原則 9×9 とする。
- 座標系は左下が原点 `(0,0)`。右方向が `+x`、上方向が `+y`。UI 表示では下段が y=0。
- 塔フロアの開始位置は `DungeonFloorDefinition.spawnPoint` で定義する。同一区間内では前フロアの階段位置を次フロアの開始位置にできる。
- 盤面の四角い枠は、このターンに実際に移動できるマスだけを示す。床落ちカード、危険範囲、ダメージ罠、ひび割れ/崩落床は枠ではなく塗りや小マーカーで表現する。

## 2. カードと所持品

- 塔ダンジョン本編では通常の手札/NEXT/山札補充を使わず、基本移動と拾得/報酬カードの回数制インベントリで進む。
- `DungeonFloorDefinition.cardPickups` のマスを踏むと、追加手数なしでカードを取得する。拾得カードは1回使い切りで、クリア時に未使用分が残っていれば次フロアへそのまま持ち越す。
- 報酬カードと持ち越した拾得カードは `DungeonRunState.rewardInventoryEntries` に保持し、区間内フロア間だけ持ち越す。
- 所持カードは通常カード9種類までにし、補助カードも通常所持枠を1種類として数える。基本移動が許可された塔では10枠目を消費されない基本移動カードとして固定表示する。同じカードは種類枠を増やさず残り使用回数として積み、使用時は拾得分から先に消費する。
- 9種類満杯の状態で未所持の床落ちカードを踏んだ場合は、拾ったカードと所持中の9種類から捨てるカードを選ぶ。拾ったカードを選ぶと取得せず盤面から消し、所持カードを選ぶとその種類の残り回数をすべて捨てて拾ったカードを取得する。同種カードを踏んだ場合は選択を挟まず自動で回数を積む。この取捨選択では追加手数やペナルティは発生しない。
- 報酬は同一区間内に次フロアがあるクリア後に選ぶ。新しい移動/補助カードを手札に追加するか、所持報酬カードを強化する。
- 手札に追加した移動カードは通常2回使用可能にする。成長スキル「カード温存」取得後は対象の使用回数を3回にする。補助カード `補給` は強力な1回報酬として扱い、カード温存の回数増加対象にしない。
- `補給` は移動せず1手使って自身を消費し、使用前に空いていた通常カード枠を、そのフロアのクリア後報酬候補プールに含まれる未所持の移動カードでランダムに補給する。補給で追加されたカードは現在フロア限りの1回カードで、クリア時に未使用分が残っていれば次フロアへそのまま持ち越す。
- `DungeonRules.allowsBasicOrthogonalMove` が true のフロアでは、10枠目の基本移動カードから上下左右1マスへ基本移動できる。基本移動はカードを消費せず、手数を1増やし、敵ターン、床ギミック、出口判定、手数切れ判定の対象になる。ガイドモードがオンの場合は従来どおり盤面に表示された基本移動候補を直接タップできる。ガイドモードがオフの場合は、基本移動カードまたは通常カードを先に選び、表示された候補マスをタップして確定する。

## 3. クリア・失敗・処理順

- 各フロアは `GameMode.CompletionRule.dungeonExit(exitPoint:)` を使い、指定出口/階段マスへ到達するとクリアする。
- レイ型など途中マスも踏むカードでは、移動途中で解錠済みの出口/階段へ入った時点で移動を止めてクリアする。
- ダンジョン失敗条件は `DungeonFailureRule` で管理する。現行では HP 0 と残り手数 0 を扱う。
- 移動後の処理順は `床ギミック → 出口判定 → 敵ターン → 失敗判定`。出口へ着いた手は、残り手数 0 や敵ターンより先にクリアとして扱う。
- フロア間では同一区間内だけ HP と所持カードの残り使用回数を引き継ぎ、手数、敵、床状態、床落ちカード配置は各フロア開始時にリセットする。
- 失敗時の再挑戦は現在の区間開始階から始める。

## 4. 塔の種類

- `DungeonLibrary.shared` は当面 `基礎塔`、`成長塔`、`試練塔` を提供する。
- 基礎塔はチュートリアル用の 5×5 / 3 フロア構成とし、出口到達、HP、手数、番兵、ひび割れ床を順に確認する。永続成長ポイント、スキルツリー効果の対象外にする。
- 成長塔は 9×9 / 20 フロアの `.growth` 本編塔とする。1-10F と 11-20F の2区間で構成し、10Fクリア後は11Fから開始できる。
- 成長塔では5F / 10F / 15F / 20Fの初回クリアで成長ポイントを各1得る。同じ区切りでは重複付与しない。
- 試練塔 v0 は 9×9 / 3 フロアの `.roguelike` 塔とし、永続成長ポイント、スキルツリー効果、塔ごとの成長獲得表示の対象外にする。
- タイトルの通常プレイ導線は塔選択だけを表示する。旧キャンペーン、Daily、ハイスコア、旧ランキング画面は通常導線に出さない。

## 5. 成長塔

- 成長塔は巡回、鍵、見える罠、ワープ床、敵警戒、回転見張り、追跡兵、予告兵、固定の岩/柱、床割れを段階的に複合する。
- 岩/柱は1フロア2〜4個程度の固定障害物で、鍵や開門では消えない。
- 1-8Fは難しいギミックを足す代わりに拾得カードを各5枚前後へ増やし、拾うほど短縮、安全ルート、鍵やカード回収への寄り道が楽になる低難度区間として扱う。
- 成長塔ではラン開始時に `DungeonRunState.cardVariationSeed` を決め、同じラン中は固定したまま、拾得カードの種類/位置とクリア後報酬候補を階ごとに解決する。
- 同一区間内では前フロアの階段位置を次フロアの開始位置にし、階段位置も複数パターンへ分散して周回時の固定感を減らす。
- ひび割れ床は区切り階や最終階など、通常遷移できる次階がない階には置かない。

## 6. 永続成長

- 成長ポイントは塔選択の「成長」で使用し、準備、報酬、危険回避の3枝型スキルツリーから取得する。
- 準備枝は「道具袋」と「登り支度」。道具袋は区間開始時に `右2` を1回分持って始める。登り支度は道具袋と10F到達後に取得でき、区間開始時に `右2` と `上2` を各1回分持って始める。
- 報酬枝は「報酬の目利き」と「カード温存」。報酬の目利きは成長塔の報酬3択へ、既存候補を壊さず補完カードを1枚混ぜる。
- 危険回避枝は「足場読み」と「踏み直し」。足場読みは成長塔の各区間で最初の罠または床崩落ダメージを1回だけ無効化する。踏み直しは足場読みと15F到達後に取得でき、区間ごとの無効化回数を2回にする。
- フロア間の HP 引き継ぎは通常どおり `DungeonRunState.carriedHP` を使い、永続成長で HP は増やさない。

## 7. 敵・床・ギミック

- 敵は `EnemyDefinition` で配置し、進行中は `EnemyState` として管理する。現行では番兵、巡回兵、見張り、回転見張り、追跡兵、予告兵を扱う。
- 巡回兵は巡回路全体を薄い線と小さな点のレールとして常時表示し、現在地から次に進むマスへの矢印をレール上に重ねて表示する。
- 回転見張りは指定された視線方向をプレイヤーの1手ごとに順番に切り替える。現在向いている方向だけを危険範囲として扱い、次に向く方向は小さな矢印で常時表示する。
- 追跡兵はプレイヤーの1手後に最短経路で1マス近づく。通行不可マス、崩落床、盤外は避け、到達不能なら動かない。危険範囲は現在地と上下左右1マスだけで、次に動くマスへの小さな矢印を常時表示する。
- 予告兵は移動せず、指定方向を順に切り替えながら短い直線を薄い警告床として表示する。警告床は現在の危険範囲とは別扱いで、次の敵ターンにそのマスへいると1ダメージを受ける。警告床は通行不可マス、崩落床、盤外で止まる。
- `HazardDefinition.damageTrap` は見えている罠床として扱い、基本移動やカード移動で入った罠マスごとに HP を減らす。カード移動の途中マスも対象になる。
- `HazardDefinition.brittleFloor` は 1 回踏むとひび割れ、2 回目で崩落して通行不可になる。崩落時は HP を 1 減らし、報酬選択や成長ポイント付与を発生させず、短い落下演出の後に開始画面を挟まず下階の同じマスへ落下する。
- `DungeonRules.exitLock` が設定されたフロアでは、指定された鍵マスを踏むまで出口/階段は施錠中として扱う。鍵は階段ロック解除専用で、取得後は床落ちカードと同じように盤面から消える。
- 成長塔と試練塔では `DungeonFloorDefinition.warpTilePairs` から `TileEffect.warp` を自動生成し、2点以上のワープ入口を循環ワープとして扱う。

## 8. 実装参照マップと更新手順

- ルール全般の実装: `Game/GameCore.swift`・`Game/GameCore+Penalty.swift`。
- カード定義と山札: `Game/MoveCard.swift`・`Game/Deck.swift`。
- モードとレギュレーション: `Game/GameMode.swift`（`Regulation`・`PenaltySettings`・`DungeonMetadata`）。
- 塔ダンジョン固有設定: `Game/DungeonDefinition.swift`。
- 塔ダンジョンの永続成長保存と適用判断: `Services/DungeonGrowthStore.swift`。
- 塔攻略の中断復帰保存: `DungeonRunResumeSnapshot` と `DungeonRunResumeStore`。保存枠は1つだけで、塔選択の「続きから」から直近の一手後へ戻る。
- 詳細な塔ごとの規定は [`docs/campaign-stage-regulations.md`](campaign-stage-regulations.md) を参照。
- 更新フロー: コード変更 → 単体テスト更新 → 本書・塔ドキュメント改訂 → 検証。
